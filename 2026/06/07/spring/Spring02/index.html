IOC

IOC 是 Inversion of Control 的缩写，中文翻译为”控制反转”。它是一种设计原则，指的是将对象的创建和依赖关系的管理交给一个容器来处理，而不是由对象自己来管理。

Spring 框架中的 IOC 容器负责创建、配置和管理应用程序中的对象。通过使用 IOC，开发者可以实现松耦合的代码结构，提高代码的可维护性和可测试性。

简单来说，当我有一个类时，如果我想要使用它，以往的情况下，我们需要自己去 new 这个类的实例，并且管理它的生命周期。而在 IOC 中，我们将这个责任交给了 Spring 容器，我们只需要在配置文件中声明这个类，Spring 就会帮我们创建它的实例，并且管理它的生命周期。我们只需要在使用的时候从容器中获取这个实例即可。

XML 配置

在 Spring 的早期版本中，IOC 配置通常是通过 XML 文件来完成的。我们可以在 XML 文件中定义 bean（即对象），并指定它们的属性和依赖关系。

除了 XML 配置，还有很多配置方式，具体则是通过 BeanDefinitionRegister 接口来实现的，其中有三个现成的实现类，分别是：XMLBeanDefinitionReader、PropertiesBeanDefinitionReader 和 AnnotatedBeanDefinitionReader。其中，XMLBeanDefinitionReader 是通过 XML 文件来配置 bean 的，而 AnnotatedBeanDefinitionReader 则是通过注解来配置 bean 的。

依赖注入

依赖注入（Dependency Injection，简称 DI）是 IOC 的一种实现方式。它指的是将对象的依赖关系通过外部配置的方式注入到对象中，而不是由对象自己来管理。

常见的依赖注入方式有三种：构造器注入、Setter 注入和字段注入。

BeanFactory

BeanFactory 是 Spring 框架中的一个核心接口，它是 IOC 容器的基础。BeanFactory 负责创建和管理应用程序中的 bean（即对象）。它提供了一个统一的接口来获取和管理 bean 的实例。

ApplicationContext

ApplicationContext 是 BeanFactory 的一个子接口，它提供了更多的功能和特性。除了 BeanFactory 提供的基本功能外，ApplicationContext 还支持国际化、事件传播、资源加载等功能。其中有三个重要的实现类，分别是：ClassPathXmlApplicationContext、FileSystemXmlApplicationContext 和 AnnotationConfigApplicationContext。

实现类 描述
ClassPathXmlApplicationContext 通过 classpath 来加载 XML 配置文件
FileSystemXmlApplicationContext 通过文件系统来加载 XML 配置文件
AnnotationConfigApplicationContext 通过注解来配置 bean
WebApplicationContext 用于 web 应用程序的 ApplicationContext
