Servlet

  1. Servlet 是 Java Web 中的一种技术，用于处理客户端的请求和响应
  2. Servlet 是 JavaWeb 三大组件之一，其他两个是 Filter（过滤器）和 Listener（监听器）

第一个 Servlet 程序

Servlet 是一个接口，因此我们需要实现 Servlet 接口，才能创建一个 Servlet 程序

注意：Servlet 接口在 JDK8 之后就不存在 jaxax.servlet.Servlet 中了，需要使用 jakarta.servlet.Servlet，或者修改 JDK 到 8 以下版本

jakarta.servlet-api

  1. 创建一个类，实现 Servlet 接口
  2. 重写 service 方法，用于处理客户端的请求和响应
  3. 编辑 web.xml 文件，将 Servlet 类名和访问路径映射起来

当浏览器访问 http://localhost:8080/工程名/api/test 时，会调用 com.codedog.servlet 包下 TestServlet 类的 service 方法

Servlet 生命周期

  1. 执行 Servlet 构造器方法
  2. 执行 init 方法
  3. 执行 service 方法
  4. 执行 destroy 方法

当启动 Tomcat 服务器后，Servlet 并不会被创建，直到浏览器访问该 Servlet 才会被创建，会先执行 构造器方法，再执行 init 方法，且只会执行一次，当每次接收到请求时，会执行 service 方法，可以多次执行，当 Tomcat 服务器关闭时，会执行 destroy 方法，

请求分发

在前边学习 Form 表单的时候我们学习过，请求方法分为 GET 和 POST 两种，而在 Servlet 中，我们只能通过 service 方法来处理请求和响应，因此，我们需要在 service 方法中判断请求方法，根据不同的请求方法，调用不同的方法来处理请求和响应

在此基础上，我们可以将 GET 方法和 POST 方法分别处理，定义 doGet 方法和 doPost 方法，在 service 方法中判断请求方法，根据不同的请求方法，调用不同的方法来处理请求和响应

简化 Servlet 的开发

前边学习了使用实现类来实现 Servlet 接口的方式创建 Servlet，除此之外，我们还可以直接继承 Servelt 的实现类，如：HttpServlet，可以简化 Servlet 的开发，是一般工程中常用的方法

使用 IDEA 生成 Servlet

  1. 右键包 -> New -> Create New Servlet
  2. 输入Servlet 别名，Servelt 所在包名，Servlet 全类名
  3. 点击 OK 按钮

Servket 继承体系

ServlecConfig 类

ServletConfig 类是一个接口，用于获取 Servlet 的配置信息，如：Servlet 的别名、初始化参数等，每一个 Servlet 都有一个 ServletConfig 对象，用于获取 Servlet 的配置信息

ServletConfig 被保存在 GenericServlet 类中，在 Servlet 中可以通过 getServletConfig() 方法获取，是 GenericServlet 在初始化时通过 init 方法保存的，因此，当我们通过继承创建 Servlet 时，如果我们重写 init 方法，必须调用 super.init(servletConfig) 方法，来确保 ServletConfig 被保存到 GenericServlet 中，否在后续我们使用 getServletConfig() 获取 servletConfig 对象会返回 null

ServletContext 类

ServletContext 类是一个接口，用于获取 Servlet 的上下文信息，且一个工程只有一个 ServletContext 对象，通过 ServletConfig 调用 getServletContext() 方法获取

用途

  1. 获取 web.xml 文件中的初始化参数
  2. 获取工程路径
  3. 获取工程在服务器上的绝对路径
  4. 可以向 Map 一样通过 setXXX 和 getXXX 方法，保存和获取数据
