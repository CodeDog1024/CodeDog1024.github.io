day 01

什么是数据库？什么是数据库管理系统？什么是 SQL ？

  1. 数据库：DataBase 简称 DB，按照一定格式存储数据的一些文件的组合。这些文件中存储了特定格式的数据
  2. 数据库管理系统：DataBaseManagement 简称 DBMS，用来管理数据库中的数据（增删改查）。常见数据库管理系统：MySQL、Oracle、MS SqlServer、DB2、sybase……
  3. SQL：结构化查询语句，程序员通过编写 SQL 语句，交给 DBMS 执行，一次来完成数据库中数据的增删改查。（在任何 DBMS 中通用 ）
  4. 三者关系；DBMS –执行–> SQL –操作–> DB

数据库的基本单元

  1. 数据库的基本单元是表：table
  2. 一张表由行（row）和列（column）组成，每一行称记录，每一列称字段

SQL 语句的分类

  1. DQL：数据查询语言，查（select）
  2. DML：数据操作语言，增（insert）、删（delete）、改（update）
  3. DDL：数据定义语言，操作表的结构，如建表（create）、删除表（drop）、修改表（alter）……
  4. TCL：事务控制语言，事务提交（commit）、事务回滚（rollback）
  5. DCL：数据控制语言，授权（grant）、撤销权限（rovoke）……

  3. where 中不能使用分组函数

分组查询

  1. 使用分组函数之前必须使用分组查询
  2. 原因：一个查询语句的执行顺序为：from -> where -> group by -> select -> order by,所以，分组函数可以在 select 后使用，不可以在 where 中使用，在 where 中使用，还没有执行到group by，所以对于程序来说，还没有分组，因此报错。
  3. 若想对分组之后的查询结果再次进行条件过滤，可以使用 having + 条件，并且 having 必须和 group by 一起使用。
  • 虽然 where 和 having 都可以进行条件查询，但是 where 效率相对较高。如果两者都可已解决问题，有限选择 where

笛卡尔积

  1. 当两张表进行连接查询时，若不加任何条件，则两个表会出现笛卡尔积现象，及查询记录总数等于两个表记录总数之积
  2. 为了避免出现笛卡尔积，在进行查询时需要添加条件（两个表的共有字段相等）
  1. 为了在编写 SQL 语句时方便且增加可读性，表名同样也可起别名
