四、控制结构

顺序控制

程序由上到下按照顺序执行

分支控制

  1. 单分支

当条件表达式为 true 时执行 {} 内的代码块

  1. 双分支

当条件表达式为 true 时执行 if 后 {} 内的代码块，反之执行 else 后 {} 的代码块

  1. 多分支

从上到下依次执行条件表达式，若为 true 则执行其后对应的代码块，否则继续向下，若都为 false ，则执行末尾的 else 对应的代码块，只有一个代码块会被执行！

  1. 分支嵌套

分支控制中嵌套分支控制，尽量不要超过三层

  1. switch分支控制

从上至下依次判断表达式的值是否与 case 后的常量相等，若相等，则执行器后对应的代码块。若都不相等，则执行 default 后的代码块

break 是结束语句，意思是跳出当前代码，若没有 break,则会发生 switch 击穿，即执行完常量1后的代码块，会继续之后常量2后的代码块，直到执行完所有的常量后的代码块或者遇到 break 才会结束。

switch 表达式的值必须是 byte,short,int,char,enum,String 类型，且 case 后常量的类型必须与表达式类型相同或者可以自动转化。

  1. for循环
  1. while循环
  1. do…while循环

先执行代码块，在进行条件判断，所以无论如何，do…while循环一定会执行一次

  1. 多重循环(循环嵌套)
  1. break

跳出当前循环，且不继续执行后边的循环，用于提前终止循环

当在多重循环中使用 break 时，可以通过给循环添加标签，从而实现跳出对应的循环

  1. continue

跳出当前循环，继续执行后续的循环

continue 和 break 相同，也可以通过标签跳出对应循环

跳出当前方法，后续代码不再执行

  4. 有两个属性 first 和 last 分别指向头节点和为节点

  1. 可以说是无序的，有特殊情况（LinkedHashSet），没有索引
  2. 不允许有重复的数据，添加重复的数据方法会返回 false ,但是不会报错。所以最多只能包含一个 null
  3. 在进行遍历的时候，采用迭代器和增强 for 循环都可以，但不可以根据索引去取值，因为没有索引

HashSet 实现类

  1. 底层实际是一个 HashMap，是 数组+链表+红黑树
  2. 元素的存储位置取决于其哈希值，所以其存储的顺序是不确定的，因此是无序的。
  3. 添加数据时会先获取其 hachcode 找到其应该存储的位置，若该位置没有数据则直接存入，若已有数据会再判断其 equals ，才能确定两者是否重复，hashcode 是根据地址所得，所以不同对象的哈希值是不同的，故即便对象存储的数据相同，也是可以重复的，但是部分引用数据类型会重写 hashcode 和 equals，如 String ，就会导致即便是不同对象，但是因为储存的数据相同而不能重复
  4. 第一次添加数据时，会将数组的容量扩大为 16*1 = 16，并且将临界值设为 16*0.75 = 12 ，当添加的数据达到临界值 12 时，会再次扩容至 16*2 = 32，同时临界值为 32*0.75 = 24，之后依此类推.
  5. 再 java8 中，当数组的大小超过 64 ，并且单个链表的长度超过 8，就会转为红黑树，由 Node 转为 TreeNode，否则仍采用上述的扩容方式

LinkedHashSet

  1. 继承自 HashSet ，底层使用的时 LinkedHashMap，时数组+双向链表
  2. 虽然也是通过 hashcode 来确定存储的位置，但是 LinkedHashSet 会使用链表来记录元素的上一节点和下一节点，以此来保证元素再集合中的顺序。所以 LinkedHashSet 时特殊的，是有序的。

四十一、反射

什么是反射

Java反射就是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意方法和属性；并且能改变它的属性

它能做什么

我们知道反射机制允许程序在运行时取得任何一个已知名称的class的内部信息，包括包括其modifiers(修饰符)，fields(属性)，methods(方法)等，并可于运行时改变fields内容或调用methods。那么我们便可以更灵活的编写代码，代码可以在运行时装配，无需在组件之间进行源代码链接，降低代码的耦合度；还有动态代理的实现等等；但是需要注意的是反射使用不当会造成很高的资源消耗

反射机制

  1. 优点：动态创建对象和适用对象，使用领过
  2. 缺点：执行效率低
  3. 解决： 关闭访问检查，调用 setAccessible 设置为 true，提高反射效率

Class 类

  1. Class 也是一个类，所以也是 Object 的子类
  2. Class 不是人为 new 出来的，而是系统创建的
  3. 对于某个类，无论采用多少种创建方式，内存中只会存在一个，创建多少个都是同一个
  4. 每个类的实例都会记得自己是由哪个 Class 实例生成的
  5. 通过一系列 API 可以获取一个类的全部信息
  6. Class 对象是保存在堆中的
  7. 类的字节码二进制文件，是放在方法区的，

事务

用于保证事物的一致性，执行多条语句时，要么全部成功，要么全部失败

事务隔离级别

脏读

一个事务读取另一个事务暂未提交的数据

不可重复读

不可重复读指的是在一个事务内，最开始读到的数据和事务结束前的任意时刻读到的同一批数据出现不一致的情况。

幻读

select 某记录是否存在，不存在，准备插入此记录，但执行 insert 时发现此记录已存在，无法插入，此时就发生了幻读。

ACID

  1. 原子性（Atomicity）：一个事务是一个不可分割的工作单元，要么都发生，要么都不发生
  2. 一致性（Consistency）：事务必须使数据库从一个一致性状态转换到另一个一致性状态
  3. 隔离性（Isolation）：当多个用户并发访问数据库时，多个事务之间互不干扰
  4. 持久性（Durability）：一个事务一旦被提交，那么它的对数据库的影响就是永久性的
