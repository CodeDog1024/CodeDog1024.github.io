MySQL 存储引擎

发表于2025-11-09|更新于2025-11-11
|浏览量:

MySQL 体系结构图

  1. 客户端连接器：Native C API、JDBC、ODBC、.NET、PHP、Python、Ruby、Cobol 等
  2. MySQL 服务端：
    • 连接层：连接池
    • 服务层：SQL 接口、解析器、查询优化器、缓存
    • 引擎层：可插拔存储引擎
    • 存储层：数据文件、索引文件、数据结构、数据结构描述文件

存储引擎简介

存储引擎是 MySQL 数据库管理系统中的核心部分，负责处理数据存储和检索。存储引擎提供了一种机制，将数据存储在数据库管理系统中，并允许用户使用各种查询语句来检索数据。存储引擎是基于表的，而不是库，所以同一个库中的不同表可以有不同的存储引擎。

  1. 查看数据库支持的存储引擎

    SHOW ENGINES;

  2. 设置存储引擎

    CREATE TABLE table_name (……) ENGINE=[存储引擎名];

存储引擎的特点

InnoDB

InnoDB 是 MySQL 默认的存储引擎，是一种兼顾高可靠性和高性能的通用存储引擎

  1. DML 操作遵循 ACID 原则、支持事务
  2. 锁机制：行级锁、表级锁、行级锁升级为表级锁
  3. 支持外键关系

在使用 InnoDB 时，每一个表数据库都会创建一个.ibd文件，该文件保存了表结构、索引、数据等信息。

innodb_file_per_table: 这个选项默认是开启的，每个表都会创建一个.ibd文件，否则，所有表数据都保存在ibdata1文件中。

逻辑存储结构

  1. Table Space：表空间，在一个 Table Space 中包含多个 Segment
  2. Segment：段，一个 Segment 包含多个 Extent
  3. Extent：区，一个 Extent 存放 64 个 Page，一个区的大小为 1M
  4. Page：页，一个 Page 存放多个 Row，一个页的大小为 16K
  5. Row：行，一个行中包含了，Trx id(最后一次操作事务的ID)、Roll pointer(指针)、字段数据

MyISAM

MyISAM 是 MySQL 早期的默认存储引擎

  1. 不支持事务和外键
  2. 支持表锁，不支持行锁
  3. 访问速度快

在使用 MyISAM 时，每一个表数据库都会创建三个文件，分别为：.MYD、.MYI、.sdi

  1. MYD: 用于存储数据
  2. MYI: 用于存储索引
  3. sdi: 用于存储表结构

Memory

Memory 是一种基于内存的临时存储引擎，用于临时存储数据，不会把数据写入到文件中，因此如果遇到断电，数据将丢失。

  1. 不支持事务和外键
  2. 访问速度更快
  3. 支持 Hash 索引

使用 Memory 时，每一个表数据库都会创建一个.sdi文件，该文件保存了表结构等信息。

存储引擎使用场景

  1. InnoDB: 用于高并发、高事务处理、数据完整性要求高的场景，适用于 90% 的场景
  2. MyISAM: 用于以读为主的场景，并且对于事务、并发性要求不高的场景，但是被 MongoDB 替代
  3. Memory: 用于临时存储数据，访问速度快，用于临时表和缓存，但是无法保证数据安全，被 Redis 替代
