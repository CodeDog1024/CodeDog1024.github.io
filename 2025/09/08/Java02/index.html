Java 语言概述

发表于2025-09-08|更新于2025-09-08
|浏览量:

什么是程序

程序：一系列有序指令的集合。

Java 诞生小故事

  • 1990 年，Sun 公司启动绿色计划
  • 1992 年，创建了 oak 语言
  • 1994 年，gosling 参加硅谷大会演示 Java 语言，震惊世界
  • 1995 年，Sun 公司发布 Java 1
  • 2009 年，Oracle 公司收购 Sun 公司
  • 2011 年，发布 Java 7

Java 技术体系平台

  • Java SE（Standard Edition）：标准版，桌面应用程序
  • Java EE（Enterprise Edition）：企业版，服务器端应用程序
  • Java ME（Micro Edition）：微型版，嵌入式设备应用程序

Java 重要特点

  1. Java 语言是面向对象的(OOP)
  2. Java 语言是健壮的，Java 是强类型语言，异常处理、垃圾回收等是 Java 程序健壮性的重要保障
  3. Java 语言是跨平台的，一次编写，到处运行
  4. Java 语言是解释型的

Java 开发工具

  • EditPlus
  • Notepad++
  • Sublime Text
  • IntelliJ IDEA
  • Eclipse

Java 运行机制及运行过程

跨平台

Java 语言之所以能够实现跨平台，主要是因为 Java 语言不是直接运行在操作系统上的，而是运行在 Java 虚拟机（JVM）上。因此，只需要为不同的平台提供不同的 JVM，就可以实现同一个 class 文件在不同平台上运行。

JVM 虚拟机

JVM（Java Virtual Machine）：Java 虚拟机，是 Java 语言的核心组成部分，负责 Java 程序的加载、解释和执行，将 Java 程序与具体的操作系统和硬件平台隔离开来，消除了因为不同平台之间的差异。

运行过程

.java → 编译器（javac） → .class → 运行（java） → JVM → 解释器（JIT） → 机器码。

什么是 JDK、JRE

  • JDK（Java Development Kit）：Java 开发工具包，包含 JRE 和 Java 开发工具（javac、javadoc、javap、javah 等）
  • JRE（Java Runtime Environment）：Java 运行时环境，包含 JVM 和 Java 核心类库
  • JVM（Java Virtual Machine）：Java 虚拟机，负责 Java 程序的加载、解释和执行
  • JRE = JVM + Java 核心类库
  • JDK = JRE + Java 开发工具
  • JDK = JVM + Java 开发工具 + Java 核心类库

配置环境变量

为什么要配置环境变量

在命令提示符中执行任何命令时，系统都会从环境变量中寻找对应的命令，如果找不到，就会提示不是内部或外部命令，也不是可运行的程序或批处理文件。因此，为了能够在命令提示符中直接使用 Java 命令，需要将 JDK 的 bin 目录添加到系统的环境变量中。

如何配置环境变量

  1. 右键此电脑 → 属性 → 高级系统设置 → 环境变量
  2. 在系统变量中添加 JAVA_HOME，指向 JDK 安装路径，例如 D:\Java\jdk1.8.0_281
  3. 在 Path 变量中添加 %JAVA_HOME%\bin
  4. 在命令提示符中输入 java -version 测试

Java 快速入门

需求

编写一个 Java 程序，打印 Hello, World!

步骤

  1. 创建一个 HelloWorld.java 文件
  2. 编写代码
  3. 使用 javac HelloWorld.java 命令编译代码
  4. 使用 java HelloWorld 命令运行程序

课堂小练习

什么是编译

编译：将高级语言（如 Java）转换为机器语言（如字节码）的过程。编译器（如 javac）负责将源代码（.java 文件）转换为字节码（.class 文件），以便 JVM 可以执行。

什么是运行

运行：将编译后的字节码（.class 文件）加载到 JVM 中，并由 JVM 解释执行，最终转换为机器码，在计算机上运行。当源文件被修改后，需要重新编译生成新的字节码文件，然后才能运行最新的代码。

Java 开发注意事项和细节说明

  1. Java 源文件的以 .java 为扩展名
  2. Java 程序的入口是 main 方法，写法固定，必须是 public static void main(String[] args)
  3. Java 是区分大小写的语言
  4. Java 语句以分号 ; 结尾
  5. 大括号成对出现
  6. Java 源文件可以包含多个类，但只能有一个类是 public 类
  7. public 类的类名必须与文件名相同
  8. main 方法必须定义在类中，可以是 public 类，也可以是非 public 类

课堂练习

易犯错误

  1. 找不到文件：确保文件名大小写一致，确保当前目录下存在该文件
  2. 公共类名与文件名不一致：Java 源文件名必须与公共类名一致
  3. 缺少分号：每一句语句必须以分号结束
  4. 非法字符：检查是否有拼写错误或非法字符，如中文符号

注释

介绍

注释：用于解释代码，增加代码的可读性，编译器会忽略注释内容。

分类

  1. 单行注释：使用 // 开头，例如：// 这是单行注释
  2. 多行注释：使用 /* 开头，使用 */ 结尾，例如：/* 这是多行注释 */
  3. 文档注释：使用 /** 开头，使用 */ 结尾，例如：/** 这是文档注释 */

代码

Java 代码规范

  1. 对于类、方法的注释，要使用文档注释，其他使用单行或多行注释
  2. 注意代码缩进
  3. 运算符两侧使用空格隔开，如：a + b
  4. 源文件使用 UTF-8 编码
  5. 一行内容不要超过 80 个字符
  6. 大括号使用行尾风格

DOS 命令

DOS 介绍

DOS（Disk Operating System）disk operating system，磁盘操作系统，是一种用于管理磁盘的计算机操作系统。通过 DOS 命令，用户可以对磁盘进行管理，如创建、删除、修改、查看文件、目录等。

相对路径和绝对路径

在 DOS 中，路径有两种类型：相对路径和绝对路径。顾名思义，相对路径是指相对于当前目录的路径，而绝对路径则指从根目录开始。
