static 静态变量

static 是静态修饰符，用于修饰成员变量、成员方法，当一个成员变量被 static 修饰后，这个成员变量被称为静态成员变量，该类所有的实例化对象共享这个变量，该变量可以通过类名直接访问，如：Student.teacherName。

static 静态方法和工具类

静态方法

同理，被 static 修饰的方法称为静态方法，使用类型即可访问，无需实例化对象，如：Math.abs()，多用于工具类中。

工具类

工具类是指一些静态方法的集合，通常不需要实例化对象即可使用。常见的工具类有：java.lang.Math、java.util.Collections等。

static 的注意事项

  1. 在静态方法内，只能访问静态变量和静态方法
  2. 在非静态方法内，既可以访问静态变量和静态方法，也可以访问非静态变量和非静态方法
  3. 静态方法中不能使用 this 关键字

继承

什么是继承

继承是面向对象三大特征之一，继承是类与类之间的关系，继承是类与类之间的一种 IS-A 关系，继承是类与类之间的一种包含关系。

继承的作用

前边已经学到了实体类，那么当多个实体类具有相同的属性和方法时，就可以将这些共同的属性和方法提取出来，定义在公共类中，这个公共类就是父类，让这些实体类继承父类，这些实体类称为子类，子类可以通过继承来获取父类所定义的属性和方法。且子类也可以添加新的属性和方法，或者覆盖（重写）父类中的方法。

继承的语法

继承的语法：public class 子类 extends 父类

继承的特点

特点

  1. 一个子类只能继承一个父类，但是一个父类可以有多个子类
  2. 继承支持多级继承，如：A extends B，B extends C，这样子 B 可以获得 C 的属性和方法，A 可以同时获取 B 和 C 的属性和方法
  3. Java 中的所有类都直接或间接继承自 Object 类，Object 类是所有类的根类

子类到底能继承父类的哪些属性和方法？

分类

分类 私有 非私有
构造方法 不能继承 不能继承
成员变量 可以继承（不能直接使用） 可以继承
成员方法 不能继承 可以继承（虚方法表）

解释说明

  1. 因为构造方法要求必须与类名相同，所以子类不能继承父类的构造方法
  2. 父类的成员变量私有属性，子类可以继承，但是不能直接访问，需要通过暴露出来的 set/get 方法进行访问
  3. Java 底层会创建一个虚方法表，从 Object 类开始，以此类推，把类中的非 private、非 static、非 final 的方法都提取到虚方法表中，子类会继承虚方法表，直至继承结束，最终的子类通过虚方法表会继承前边所有类的方法

继承中成员变量和方法的访问特点

成员变量

当子类和父类存在同名成员变量，访问变量时，遵循就近原则，即：局部变量 > 子类成员变量 > 父类成员变量。若想指定访问子类变量，前边学过使用 this 关键字，而访问父类变量则使用 super 关键字

成员方法

与成员变量同理，在此基础上，若子类的方法和父类的方法重名，这种情况称为方法的重写，主要用于当父类方法不能满足需求时，进行方法重写，子类方法会替换掉父类方法，还需要再添加@Override 注解，表示该方法是重写父类的方法。

重写注意事项

  1. 重写方法的名称和形参列表必须和父类方法一致
  2. 访问权限修饰符必须大于等于父类方法（private < default < protected < public）
  3. 返回值类型必须和父类方法一致，或者子类方法返回值类型是父类方法返回值类型的子类

构造方法

子类无法继承父类的构造方法，但是子类在执行自己的构造方法时，会在自己的构造方法中隐式调用父类的构造方法，然后再执行构造方法中的后续代码。因为子类需要使用父类中的数据。如果父类没有完成初始化，子类无法使用。当然也可以通过 super() 显示调用父类的构造方法，且必须在第一行调用。

this 和 super

  1. this 关键字，表示当前对象
  2. super 关键字，表示父类对象

多态

什么是多态

多态是 Java 中一个很重要的概念，它允许对象具有多种形态。多态性允许对象执行不同的行为，这取决于对象所属的类。

表现形式

在由继承关系的前提下，父类引用指向子类对象，如：Persion person = new Student();，这时父类引用对象会调用子类对象中的属性和方法，这个过程就叫多态性。

特点

解释说明

子类在继承父类时，获取了父类的属性和方法，虽然子类存在同名的属性，但是并不会覆盖父类的属性，所以在多态中，因为数据类型是父类的类型，所以在访问属性时，会访问父类的属性，但是子类中与父类同名的方法则会覆盖虚方法表中父类的方法，所以通过多态访问方法时，会调用子类的方法。只有子类中没有重写父类的方法时，才会调用父类的方法。

多态的优势和弊端

  1. 解耦合：多态性允许对象具有多种形态，这允许对象在运行时改变其类型，从而实现解耦合
  2. 参数兼容性：多态性允许对象具有多种形态，这允许对象在运行时改变其类型，从而实现参数兼容性
  3. 当把子类对象赋给父类引用时，那么该对象就无法再调用子类特有的方法，因为编译时是参照父类类型，此时就需要用到向下转型，也就是把父类对象转换成子类对象，如：Student student = (Student) person;，此时需要用到强制类型转换

包和 final

包

包的本质就是一个文件夹，用于对不同的 Java 类进行分类存储，方便管理，包名的命名应使用域名倒序的方式，如：com.atguigu.java，且全部小写，见名知意。

每个 Java 文件都需要在文件头声明包名，如：package com.atguigu.java;，在使用其他包中的类时，需要使用全限定名，如：com.atguigu.java.Person，但是尽量避免使用全限定名，使用 import 引入包中的类，如：import com.atguigu.java.Person;，这样在使用 Person 类时就可以使用 Person，而不需要使用全限定名。

包的注意事项

  1. 使用同一包内的类时，或者 java.lang 包中的类时，不需要导入包
  2. 使用同名类时，需要使用全限定名

final

final 关键字用于修饰类、方法、变量，修饰类时，表示该类不能被继承，修饰方法时，表示该方法不能被重写，修饰变量时，表示该变量不能被修改。

接口

接口的用途

当一个子类继承一个抽象类时，必须实现抽象类中所有的抽象方法。可是如果该子类并不具备该行为时，就无法重写，比如：在 Animal 中定义了一个 fly 抽象方法，对于 Bird 来说，它具备 fly 行为，可以重写，但是对于 Dog 来说，它不具备 fly 行为，无法重写，可是又不得不重写（编译报错），因此，接口诞生了。接口是一种抽象的类型，它定义了行为，但是不提供具体实现，需要类来实现接口并提供具体实现。

接口的定义

使用 interface 关键字来定义接口，如：public interface 接口名 {...}，使用时，使用implements关键字来实现接口，如：class 类名 implements 接口名 {...}，与抽象类不同，同一个类可以实现多个接口。

接口的代码示例

接口中成员的特点

成员变量

接口中的成员变量都是 public static final 的，并且只能是常量。

构造方法

接口中不能有构造方法。

成员方法

JDK 1.7 以前，接口中的成员方法都是抽象方法，JDK 1.8 时，接口中的成员方法可以时普通方法，JDK 1.9 时，接口中可以定义私有方法。

接口和类的区别

  1. 一个类只能继承一个类，但是可以实现多个接口
  2. 接口之间也可以继承，且可以继承多个接口

接口知识扩展

新特性

JDK 8 中接口中可以定义有方法体的方法，分为默认方法（default）、静态方法（static），对于默认方法，实现类可已选择是否重写，但是当多个接口含有重名的默认方法时，实现类必须重写该方法。对于静态方法，实现类无法重写。JDK 9 中接口中可以定义私有方法。

新特性代码示例

适配器设计模式

当一个接口中有 N 个方法，但是实现类只需要其中某几个方法时，若用该类实现接口，则必须重写全部方法，此时可以使用适配器模式，创建一个适配器类，实现该接口，并重写其中的方法，但是不要编写方法体，然后再让对应的类继承适配器类，这样，子类只需实现需要使用的方法即可。

内部类

什么是内部类

内部类是定义在另一个类里面的类。内部类是一个独立的个体，但是又依附于外部类的存在。内部类可以直接访问外部类的属性和方法，但是外部类若想访问内部类，则需要先实例化内部类对象，通过对象才可以访问。

分类

  1. 成员内部类（了解）
  2. 静态内部类（了解）
  3. 局部内部类（了解）
  4. 匿名内部类（掌握）

成员内部类

写在类中，与成员变量和方法同级，所以也可以被访问权限修饰符修饰，在 JDK 16 开始，在内部类中可以定义静态变量。当内部类和外部类出现同名属性时，采用就近原则，也可以使用 this 关键字访问内部类自己的属性。使用 外部类类名.this 访问外部类自己的属性。

静态内部类

被 static 修饰的内部类称为静态内部类，静态内部类的特性同静态成员变量和静态成员方法一样，在静态内部类中，只能访问静态属性和方法，其他属性和方法则需要实例化外部类对象才可以访问。

局部内部类

局部内部类是在方法内部定义的类，与局部变量同级，外部无法访问局部内部类，只能在所在方法内访问局部内部类。亦或者通过方法返回局部内部类对象。

匿名内部类

当有一个抽象类或者接口时，如果我们想调用其中的方法，就必须先创建一个类用于继承和实现，然后再重写对应的方法，此时就可以使用匿名内部类，其实就相当于跳过了创建类的过程。
