static 静态变量

static 是静态修饰符，用于修饰成员变量、成员方法，当一个成员变量被 static 修饰后，这个成员变量被称为静态成员变量，该类所有的实例化对象共享这个变量，该变量可以通过类名直接访问，如：Student.teacherName。

static 静态方法和工具类

静态方法

同理，被 static 修饰的方法称为静态方法，使用类型即可访问，无需实例化对象，如：Math.abs()，多用于工具类中。

工具类

工具类是指一些静态方法的集合，通常不需要实例化对象即可使用。常见的工具类有：java.lang.Math、java.util.Collections等。

static 的注意事项

  1. 在静态方法内，只能访问静态变量和静态方法
  2. 在非静态方法内，既可以访问静态变量和静态方法，也可以访问非静态变量和非静态方法
  3. 静态方法中不能使用 this 关键字

继承

什么是继承

继承是面向对象三大特征之一，继承是类与类之间的关系，继承是类与类之间的一种 IS-A 关系，继承是类与类之间的一种包含关系。

继承的作用

前边已经学到了实体类，那么当多个实体类具有相同的属性和方法时，就可以将这些共同的属性和方法提取出来，定义在公共类中，这个公共类就是父类，让这些实体类继承父类，这些实体类称为子类，子类可以通过继承来获取父类所定义的属性和方法。且子类也可以添加新的属性和方法，或者覆盖（重写）父类中的方法。

继承的语法

继承的语法：public class 子类 extends 父类

继承的特点

特点

  1. 一个子类只能继承一个父类，但是一个父类可以有多个子类
  2. 继承支持多级继承，如：A extends B，B extends C，这样子 B 可以获得 C 的属性和方法，A 可以同时获取 B 和 C 的属性和方法
  3. Java 中的所有类都直接或间接继承自 Object 类，Object 类是所有类的根类

子类到底能继承父类的哪些属性和方法？

分类

分类 私有 非私有
构造方法 不能继承 不能继承
成员变量 可以继承（不能直接使用） 可以继承
成员方法 不能继承 可以继承（虚方法表）

解释说明

  1. 因为构造方法要求必须与类名相同，所以子类不能继承父类的构造方法
  2. 父类的成员变量私有属性，子类可以继承，但是不能直接访问，需要通过暴露出来的 set/get 方法进行访问
  3. Java 底层会创建一个虚方法表，从 Object 类开始，以此类推，把类中的非 private、非 static、非 final 的方法都提取到虚方法表中，子类会继承虚方法表，直至继承结束，最终的子类通过虚方法表会继承前边所有类的方法

继承中成员变量和方法的访问特点

成员变量

当子类和父类存在同名成员变量，访问变量时，遵循就近原则，即：局部变量 > 子类成员变量 > 父类成员变量。若想指定访问子类变量，前边学过使用 this 关键字，而访问父类变量则使用 super 关键字

成员方法

与成员变量同理，在此基础上，若子类的方法和父类的方法重名，这种情况称为方法的重写，主要用于当父类方法不能满足需求时，进行方法重写，子类方法会替换掉父类方法，还需要再添加@Override 注解，表示该方法是重写父类的方法。

重写注意事项

  1. 重写方法的名称和形参列表必须和父类方法一致
  2. 访问权限修饰符必须大于等于父类方法（private < default < protected < public）
  3. 返回值类型必须和父类方法一致，或者子类方法返回值类型是父类方法返回值类型的子类

构造方法

子类无法继承父类的构造方法，但是子类在执行自己的构造方法时，会在自己的构造方法中隐式调用父类的构造方法，然后再执行构造方法中的后续代码。因为子类需要使用父类中的数据。如果父类没有完成初始化，子类无法使用。当然也可以通过 super() 显示调用父类的构造方法，且必须在第一行调用。

this 和 super

  1. this 关键字，表示当前对象
  2. super 关键字，表示父类对象

多态

什么是多态

多态是 Java 中一个很重要的概念，它允许对象具有多种形态。多态性允许对象执行不同的行为，这取决于对象所属的类。

表现形式

在由继承关系的前提下，父类引用指向子类对象，如：Persion person = new Student();，这时父类引用对象会调用子类对象中的属性和方法，这个过程就叫多态性。

多态中调用成员变量和方法的特点

特点

  • 变量：编译时看父类，运行时看父类
  • 方法：编译时看父类，运行时看子类

解释说明

子类在继承父类时，获取了父类的属性和方法，虽然子类存在同名的属性，但是并不会覆盖父类的属性，所以在多态中，因为数据类型是父类的类型，所以在访问属性时，会访问父类的属性，但是子类中与父类同名的方法则会覆盖虚方法表中父类的方法，所以通过多态访问方法时，会调用子类的方法。只有子类中没有重写父类的方法时，才会调用父类的方法。

多态的优势和弊端

  1. 解耦合：多态性允许对象具有多种形态，这允许对象在运行时改变其类型，从而实现解耦合
  2. 参数兼容性：多态性允许对象具有多种形态，这允许对象在运行时改变其类型，从而实现参数兼容性
  3. 当把子类对象赋给父类引用时，那么该对象就无法再调用子类特有的方法，因为编译时是参照父类类型，此时就需要用到向下转型，也就是把父类对象转换成子类对象，如：Student student = (Student) person;，此时需要用到强制类型转换
