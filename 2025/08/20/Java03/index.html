数字相加

两数相加时，若数据类型不一致，需要先统一数据类型，这就需要类型转换。

类型转换的分类

  1. 隐式转换：取值范围小的数据类型转为取值范围大的数据类型。
  2. 强制转换：取值范围大的数据类型转为取值范围小的数据类型。
  3. byte < short < int < long < float < double。
  4. byte、short、char 这三种数据类型在计算时，先转为 int 类型，再做计算。

隐式转换

自动转换，无需手动转换。

强制转换

格式：(目标数据类型) 原数据，如：int a = (int)10.0;。当原数据类型的值超出转换后数据类型的取值范围时，数据会发生错误。

字符串相加

当有字符串参与了加法的执行，那么就会变成拼接而不是相加，如："123" + 123 = "123123"。

数值拆分优化

字符相加

已知，char 类型在参与计算式，会优先转换为 int 类型，且前边学习了 ASCII 码表，记录了字符与数值之间的对应关系。所以当有字符参与计算时，实际上是以其 ASCII 码值进行计算的。如：'a' + 'b'，其实就是 97 + 98。

什么是逻辑运算符

在数学中，在表达多个条件的时候，我们通常会使用并且、或者等逻辑关系来讲多个条件连接起来。在 Java 中，同样也可以使用逻辑运算符来将多个条件表达式连接。

逻辑运算符分类

符号 作用 说明
& 逻辑与 并且，两边都为 true，结果为 true
| 逻辑或 或者，两边都为 false，结果为 false
^ 逻辑异或 异或，两边相同，结果为 false
! 逻辑非 取反，

短路逻辑运算符

什么是短路逻辑运算符

短路逻辑运算符的计算方法同逻辑运算符一致，无任何区别，但是在其基础上增加了短路操作，即当结果已经确定时，不再执行后续的代码。例如：在计算逻辑与时，已知左侧为 false，那么无论右侧是什么，结果都是 false，因此无需再执行右侧的代码，这时就可以使用短路逻辑运算符来增加代码执行效率。

短路逻辑运算符分类

符号 作用 说明
&& 短路与 并且，两边都为 true，结果为 true
|| 短路或 或者，两边都为 false，结果为 false

多学一招-原码、反码、补码

什么是原码、反码、补码

  • 原码：十进制数据的二进制表现形式，最左侧时符号位，0为正，1为负。
  • 反码：正数的反码和原码相同，负数的反码需要将原码除符号位以外全部取反
  • 补码：正数的补码和原码相同，负数的补码是再反码的基础上加一

总结

  • 数据在计算机都是以二进制补码的形式存储的，在运算时，也是使用补码的形式参与运算，得到的结果也同样是补码。
  • 因为补码在反码基础上加了 1，所以补码的最小值是 -128 (1000 0000)，该补码没有对应的反码和原码。
