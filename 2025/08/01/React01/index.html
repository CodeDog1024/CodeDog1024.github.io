JSX 基础-概念和本质

什么是 JSX

  • JSX 是一种 JavaScript 的语法扩展，允许我们在 JavaScript 代码中编写类似 HTML 的语法。

JSX 的本质

  • JSX 并不是标准的 JavaScript 语法，而是基于 JavaScript 的语法扩展，浏览器本身并不能识别 JSX 语法，需要通过编译工具（如 Babel）将 JSX 转换为标准的 JavaScript 代码，再进行执行。

JSX 基础-识别 JS 表达式

  • 在 JSX 中，可以使用大括号 {} 来嵌入 JavaScript 表达式。这些表达式可以是变量、函数调用或任何有效的 JavaScript 代码。

JSX 基础-实现列表渲染

  • 在 JSX 中，可以使用 JavaScript 的数组方法（如 map）来渲染列表。通过遍历数组，可以生成一组 JSX 元素。
  • 在渲染列表时，需要为每个元素提供一个唯一的 key 属性，以帮助 React 识别哪些元素发生了变化、添加或删除。

JSX 基础-实现基础条件渲染

  • 在 JSX 中，可以使用 JavaScript 的条件语句（如&&、三元运算符等）来实现条件渲染。根据不同的条件，可以选择性地渲染某些元素。

JSX 基础-实现复杂条件渲染

  • 在 JSX 中，可以使用函数来实现更复杂的条件渲染逻辑。通过将条件判断封装在函数中，可以使代码更加清晰和可维护。
  • 也可以在 JSX 中直接插入匿名函数来进行条件渲染，但是不推荐使用，及影响效率又影响代码阅读

React 组件基础使用

组件是什么

  • 一个组件是一个独立的、可复用的 UI 单元，它可以包含自己的状态和生命周期方法。组件可以是函数组件或类组件。

React 组件

  • 在 React 中，一个组件其实就是一个函数，返回一个 JSX 元素，使用的时候就像使用 HTML 标签一样。
  • 组件名必须以大写字母开头，以区分 HTML 标签和自定义组件。
  • 组件可以是单标签或双标签，单标签组件不需要闭合标签。

useState 基础使用

  • useState 是 React 中的一个 Hook，用于在函数组件中添加状态管理功能。它允许我们在函数组件中使用状态，而不需要将组件转换为类组件。
  • 被 userState 创建的状态变量可以在组件中实时被读取和更新。
  • useState 是一个函数，调用该函数会返回一个数组，第一个元素是当前状态值，第二个元素是用于更新状态的函数。

useState 修改状态的规则

  • 在 React 中，状态是不可变的，只能通过赋值函数来更新状态，比如 count++ 是错误的，count 的值虽然会被修改，但是不会触发页面的更新，需要使用 setCount(count + 1) 来更新状态。

基础样式控制

行内样式（不推荐）

  • 通过给 style 属性传入一个对象来设置样式，格式为 {属性名: 属性值}，属性名使用驼峰命名法。

外部样式表

  • 在外部创建对应的 CSS 文件，然后通过 import './XXX.css' 语法在组件中导入该文件，再通过 className='类名' 属性来应用样式。

lodash

  • lodash 是一个 JavaScript 实用工具库，提供了许多常用的函数和方法，可以帮助我们更方便地处理数组、对象、字符串等数据类型。

classnames

  • classnames 是一个用于条件地合并 CSS 类名的库，可以根据条件动态地添加或删除类名，使得样式的管理更加灵活和简洁。
