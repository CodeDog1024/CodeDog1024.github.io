React 中获取 DOM

  • 使用 useRef 来获取 DOM 元素的引用，先使用 useRef 创建一个引用对象，然后使用 ref 属性将其绑定到需要获取的 DOM 元素上然后通过 current 属性访问该 DOM 元素

处理 id 和时间

id 可以使用 uuid 库生成

  • 安装 npm install uuid

时间可以使用 dayjs 库处理

  • 安装 npm install dayjs

清空内容和聚焦实现

  • 清空内容可以使用 useRef 来获取输入框的引用，然后调用 current.value = '' 来清空内容或者直接使用 setComment('') 来清空状态聚焦可以使用 inputRef.current.focus() 来实现

组件通信-父传子-基础实现

  • 组件通信就是在 React 中不同组件之间传递数据和事件
  • 当一个组件内嵌套了另一个组件时，此时该组件称为父组件，内嵌的组件称为子组件
  • 父组件可以通过 props 将数据和事件传递给子组件，子组件可以通过 props 接收父组件传递的数据和事件

组件通信-父传子-props 说明

  1. props 可传递任意类型的数据
  2. props 是只读的，其中的数据不能被修改

组件通信-父传子-children

  • 父组件在给子组件传递参数时，若将参数编写在子组件的标签内，则该参数会被自动传递给子组件的 props.children 属性
  • children 是一个数组，包含了子组件标签内的所有内容

父子组件通信-子传父实现

  • 子组件可以通过调用父组件提供的函数来实现子向父的通信

使用状态提升实现兄弟组件通信

  • 兄弟组件就是指在同一个父组件下的两个子组件
  • 兄弟组件之间不能直接通信，但可以通过父组件来实现，子A -> 父组件 -> 子B

使用 context 机制跨层传递数据

  • 跨层传递数据是指在组件树中，某个组件的子孙组件需要访问祖先组件的数据
  • React 提供了 context 机制来实现跨层传递数据
  • 在顶层组件外使用 createContext 创建一个上下文对象，然后使用 对象.Provider 标签包裹需要共享数据的组件，最后在子孙组件中使用 useContext 钩子来访问这些数据

useEffect-概念理解和基础使用

  • useEffect 是 React 中的一个钩子函数，用于处理 React 组件中因渲染而产生的事件，也成为副作用
  • useEffect 可以在组件渲染后执行操作，比如数据获取、订阅、手动更改 DOM 等
  • 相当于 Vue.js 中的 watch 和 mounted 生命周期
  • useEffect 接收两个参数，第一个参数是一个函数，第二个参数是一个依赖数组

useEffect-不同依赖项说明

  • useEffect 的第二个参数是一个依赖数组，用于控制 useEffect 的执行时机
依赖项 说明
不传 组件初始渲染时、组件更新时都会执行
空数组 组件初始渲染时执行一次
有依赖项 组件初始渲染时执行一次，依赖项变化时执行

不传依赖项

传递空数组

传递依赖项

useEffect-清楚副作用

  • 副作用就是上述提到的 useEffect 中的操作，比如数据获取、订阅、手动更改 DOM 等
  • 清楚副作用就是在组件卸载时或者依赖项变化时，执行一些清理操作，比如取消订阅、清除定时器等
  • 相当于 Vue.js 中的 beforeDestroy 生命周期
  • useEffect 可以返回一个函数，这个函数会在组件卸载时或者依赖项变化时执行

自定义 Hook 实现

  • 自定义 Hook 的命名规则是以 use 开头, 用于将组件逻辑抽离成可复用函数

ReactHooks 使用规则

  1. 只能在组件中或者自定义 Hook 中使用 Hook
  2. 只能在顶层调用 Hook，不能在循环、条件语句或其他函数中调用 Hook
