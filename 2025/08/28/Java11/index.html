String 概述

String 是引用数据类型，所以 String 对象需要通过 new 关键字创建，如：String str = new String("hello world");，但是我们之前都是 String str = "hello world"; 创建的，是因为 java 对此做了优化。String 对象的值是不可修改的，即便使用 str = "abc";，也只是创建了一个新的字符串覆盖了原来的字符串。

直接赋值

当我们创建字符串时，JVM 会先检查字符串常量池中是否已经存在相同内容的字符串，如果存在，则直接返回该字符串的引用；如果不存在，则在堆中创建一个新的字符串对象，并将其引用放入常量池中。

new 关键字

只要是通过 new 关键字创建的，就会在堆中创建一个新的字符串

字符串相关类的底层原理

字符串存储的内存原理

  • 通过直接赋值创建的字符串会优先检查字符串常量池是否存在该字符串，有则复用，无则创建新的字符串对象。
  • 通过 new 关键字创建的字符串对象会在堆内存中分配空间，不会使用字符串常量池。

== 号比较的是什么

  • == 对于基本数据类型比较的是值
  • == 对于引用数据类型比较的是地址值，字符串之间比较应使用 equals() 方法。

字符串拼接的底层原理

  • 字符串在拼接时会做一些优化，如 String str = "a" + "b" 会被优化为 String str = "ab"。
  • 若右侧的字符串拼接有其他变量参与，则会创建新的字符串，不会优化。

StringBuilder 提高效率原理图

  • StringBuilder 的默认长度为 16，当字符数量超过 16 时，会自动扩容，新的容量为原来的 2倍 + 2。若添加的字符数量超过扩容后的容量，并不会进行两次扩容，而是直接以添加的字符数量为基准扩容。
