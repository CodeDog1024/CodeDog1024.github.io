人机交互-图形化界面的小故事

  1. 最初的计算机并没有图形化的操作界面，只有一个可以输入命令的黑框(终端)，用户需要通过输入命令来操作计算机完成对应的操作。
  2. 而后，施乐发布了世界上第一款具有图形化界面的计算机。
  3. 截至至今，终端仍然被保留在计算机中，如 Windows 中的 CMD(命令提示符)，Linux 和 MacOS 中的 Terminal。

  2. 输入 cmd，点击确定

练习-利用 CMD 打开 QQ 并配置环境变量

利用 CMD 打开 QQ

  1. 找到 QQ 的安装目录，如 C:\\Program Files (x86)\\Tencent\\QQ。
  2. 打开 CMD 并进入 QQ 的安装目录，cd C:\\Program Files (x86)\\Tencent\\QQ。
  3. 运行 QQ.exe, QQ.exe。

配置环境变量

上面的案例，必须要在 QQ 的安装目录下运行，否则会找不到 QQ.exe，若想在任意位置运行 QQ，则需要配置环境变量。将 QQ.exe 的路径添加到环境变量中，如 C:\\Program Files (x86)\\Tencent\\QQ，那么在执行 QQ.exe 时，系统会优先到环境变量中查找，找到后就会打开 QQ。

如何配置环境变量

  1. 右键 此电脑，选择 属性。
  2. 选择 高级系统设置，选择 环境变量。
  3. 在 系统变量 中，找到 Path，点击编辑。
  4. 点击新建，在 Path 中添加 C:\\Program Files (x86)\\Tencent\\QQ，点击确定保存。

环境变量

为什么安装 JDK 后不需要配置环境变量

新版本的JDK安装程序，在安装时会自动将 javac、java、javaw、jshell这四个命令添加到环境变量中。如果我们需要使用其他命令，则需要手动配置环境变量。

手动配置环境变量

  1. 打开系统环境变量
  2. 新建系统变量：Java_Home:C:\Program Files\Java\jdk1.8.0_281
  3. 选择PATH变量，点击编辑
  4. 添加%Java_Home%\bin

Java 跨平台的原理

Java 程序并不是运行在操作系统的，而是运行在 Java 虚拟机（JVM）上。Java 虚拟机是一个中间层，它将 Java 程序运行在操作系统之上。Java 虚拟机可以运行在任何操作系统上，只要它支持 Java 语言。所以即便是不同的操作系统，也可以运行同一个 Java 程序。

JRE 和 JDK

  • JRE（Java Runtime Environment）是 Java 运行时环境，它包含了 Java 运行所需的所有组件。JRE 可以在 Windows、Linux、MacOS 等操作系统上运行 Java 程序。
  • JDK（Java Development Kit）是 Java 开发工具包，它包含了 JRE 和 Java 编译器。JDK 可以在 Windows、Linux、MacOS 等操作系统上开发 Java 程序。
