Java 内存区域详解

Java 程序运行时，内存主要分为以下几个区域，每个区域有不同的作用：

区域名称 作用说明
栈（Stack） 存放方法的局部变量、方法调用信息、返回地址等。每个线程独享，生命周期与线程一致。
堆（Heap） 存放所有对象实例和数组，由垃圾回收器管理。所有线程共享，生命周期与 JVM 一致。
方法区（Method Area） 存放类的结构信息（类元数据）、静态变量、常量池等。所有线程共享。
本地方法栈（Native Method Stack） 为本地方法服务，存放本地方法调用信息。每个线程独享。
程序计数器（Program Counter Register） 记录当前线程执行的字节码行号指示器。每个线程独享。

说明

  • 栈：每次方法调用都会在栈中分配一块空间，方法结束后自动释放。
  • 堆：所有通过 new 创建的对象和数组都存放在堆中，垃圾回收器负责回收无用对象。
  • 方法区：用于存储已被虚拟机加载的类信息、常量、静态变量等数据。
  • 本地方法栈：支持 Java 调用本地（如 C/C++）方法。
  • 程序计数器：帮助线程切换和恢复执行位置。

这些区域共同协作，保证 Java 程序的高效运行和内存管理。

数组的内存图

因为数组时引用数据类型，所以数组变量保存的并不是真实的值，而是该数组在堆中的地址。真实的值保存在堆中。然后通过索引来访问具体的值。当将 A 数组 的值赋给 B 数组时，B 数组保存的只是 A 数组堆中的地址，而不是 A 数组中的值，那么此时 AB 数组就共享一个地址。当改变 A 数组中的某个值时，B 数组中的值也会改变。
