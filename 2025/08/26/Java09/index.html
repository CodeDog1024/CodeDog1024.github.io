面向对象介绍

面向对象编程（Object-Oriented Programming，简称 OOP）是一种编程范式，它使用“对象”来设计应用程序和计算机程序。它利用了抽象、封装、继承和多态这些概念。

类和对象

什么是类和对象

  1. 类：类是创建对象的模板，类中定义了对象的属性和方法。
  2. 对象：通过类创建的实例，对象是类的实例化。

类的定义

使用 class 关键字定义类，如：class Person 创建了一个名为 Person 的类，一个类由五部分组成，分别是：成员变量、成员方法、构造方法、代码块和内部类。

注意事项

  1. 类名必须大写开头，主类名(被 publie 修饰)必须与文件名一致。
  2. 一个文件中可以定义多个类，但是只能由一个类是主类。

创建对象

在定义好类之后，就可以通过 new 关键字创建对象，如：Phone phone = new Phone();，之后，便可以通过 phone.属性 或者 phone.方法() 来访问对象中的属性和方法。`

封装

什么是封装

封装（Encapsulation）是 Java 类的内部机制，它把数据（属性）和行为（方法）封装在一起，并隐藏了实现细节。

封装的好处

在上边的的例子中，我们在创建对象后，可以直接通过 对象.属性名 的方式访问属性，如：student1.age = 18;，但是这样会产生一个问题，那就是没有对数据进行校验，任何值都可以被赋给 age 属性。非常不安全，封装的思想就是：将属性通过 private 修饰符封装起来，不允许外部直接访问，而是通过get、set 方法来访问属性。

private

private 同 public 一样，都是访问权限修饰符，可以用来修饰属性和方法。被 private 修饰的属性或方法，只能在当前类中访问。外部类中不能访问。

就近原则和 this 关键字

就近原则

类中的变量称为成员变量，方法中的变量称为局部变量。当成员变量和局部变量重名时，优先访问局部变量。若想访问成员变量，可以使用 this 关键字。

this 关键字

this 关键字，可以访问当前对象的成员变量和成员方法。当成员变量和局部变量重名时，可使用 this 访问成员变量。

构造方法

什么是构造方法

在创建对象时，会调用该对象的构造方法。通过构造方法，可以在创建对象时，给对象的属性赋值。每个类都隐含一个无参的构造方法。

注意事项

  1. 构造方法名称必须与类名相同。
  2. 构造方法没有返回值类型和返回值。
  3. 构造方法要么不写，要么写两个：无参构造方法、带参构造方法。

标准 JavaBean 类

JavaBean 类，也叫 JavaBean 对象，是一种 Java 类。JavaBean 类的属性必须使用 private 修饰符修饰，并且提供 getter 和 setter 方法。JavaBean 类至少要有两个构造方法：无参构造方法、带参构造方法。

步骤

  1. 加载类：JVM 首先会检查类是否已加载，如果没有，则加载类的字节码到内存，并进行类的初始化。
  2. 分配内存：为新对象分配堆内存空间，保证对象的属性有存储位置。
  3. 初始化属性：将成员变量初始化为默认值（如 int 为 0，引用类型为 null）。
  4. 执行构造方法：调用构造方法，对对象进行进一步初始化（如设置属性值）。
  5. 返回对象引用：构造完成后，将对象的引用返回给变量。

对象赋值

对象是引用数据类型，所以对象赋值时，只是复制了对象的引用，而不是复制对象。此时两个对象指向同一个内存空间。其中的属性值是共享的。
