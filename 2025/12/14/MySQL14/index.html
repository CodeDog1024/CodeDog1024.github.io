错误日志

错误日志是 MySQL 中最重要的日志之一，它记录了 MySQL 启动时、运行中、停止时发生任何严重错误的相关信息。当数据库发生任何问题而无法使用时，应首先、立即查看此日志。

错误日志默认是开启状态，默认存放的目录为 /var/log，文件名为 mysqld.log，也可以通过 log_error 环境变量查看具体文件位置。

二进制日志

二进制日志（BINLOG）中记录了所有的 DDL 和 DML 语句，但没有 DQL 语句。

二进制日志默认是开启状态，可以通过 log_bin 环境变量查看文件具体位置。

二进制日志作用：

  1. 灾难时的数据恢复
  2. MySQL 的主从复制

二进制日志在存储时有多种格式，每种格式存储的信息不同，可以通过 binlog_format 环境变量查看。

格式 含义
STATEMENT 基于 SQL 语句的日志记录，记录的是 SQL 语句，对数据进行修改的 SQL 都会记录在日志文件中
ROW 基于行的日志记录，记录的是每一行的数据变更（默认）
MIXED 混合了 STATEMENT 和 ROW 两种格式，默认采用 STATEMENT，在特殊情况下会切换为 ROW 模式

二进制日志删除：

指令 作用
reset master 删除全部日志，重置日志编号
purge master logs to ‘binlog.xxxxxx’ 删除 xxxxxx 编号之前的所有日志
purge master logs before ‘yyyy-mm-dd hh24:mi:ss’ 删除指定时间之前的所有日志

也可以通过在 MySQL 配置文件中通过 binlog_expire_logs_seconds 环境变量来控制二进制日志过期时间来自动删除。

查询日志

查询日志中记录了所有的 SQL 语句，但是查询日志默认是不开启的，需要通过 general_log 环境变量来开启。以及通过 general_log_file 来查看文件位置。

慢查询日志

慢查询日志中记录了所有执行时间超过阈值的 SQL 语句，这个阈值可以通过 long_query_time 环境变量来设置，默认为 10，最小为 0，单位是 秒，精度可以到 微秒。并且慢查询日志也是默认关闭的，需要通过 slow_query_log 环境变量来开启。

默认情况下，慢查询日志不会记录管理语句和未使用索引的查询语句，但是我们可以通过 log_slow_admin_statements 和 log_queries_not_using_indexes 来修改开启。
