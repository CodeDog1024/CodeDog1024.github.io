JDBC 批量插入

发表于2025-12-19|更新于2025-12-19
|浏览量:

批量插入

在增删改中，对于删除、修改其本身就支持批量操作，而插入则需要多次执行。

优化

上面的批量插入，使用了 PreparedStatement，相比于使用 Statement，性能已经提升了，但是本质还是一条一条插入，在此基础上，我们可以将 SQL 分组，然后按组执行，减少与 MySQL 的交互，以此提升效率，建议 1000 条一组。

MySQL 默认不支持 Batch 操作，需要通过在 url 后边添加参数 rewriteBatchedStatements=true 来开启。

再优化

可以看到，通过 Batch 操作已经让批量插入的时间减少为原来的 1/4 了，但是仍有优化空间，但是微乎其微，因此为了效果冥想，需要将插入条数修改为 1000000 条。

在 MySQL 中，执行 SQL 时事务是默认提交的，因此每次执行 SQL 都做了开启事务、提交事务的操作，但 SQL 过多时，也会对性能有一定损耗，因此，我们可以关闭事务自动提交，等待 1000000 条 SQL 全部执行完毕后手动提交事务，较少频繁开启事务的性能损耗。
