集合的体系结构

集合分为两大类：Collection 和 Map，Collection 为单例集合，其中只能保存值，而 Map 为双列集合，保存键值对。

Collection

Collection 接口是所有单例集合的根接口，常用的实现类有 List、Set 和 Queue。

  • List：有序可重复的集合，常用实现类有 ArrayList 和 LinkedList。
  • Set：无序不可重复的集合，常用实现类有 HashSet 和 TreeSet。
  • Queue：用于存储待处理元素的集合，常用实现类有 LinkedList 和 PriorityQueue。

常用方法

方法名 描述
add 添加元素
clear 清空集合
remove 删除元素
contains 判断是否包含某个元素
isEmpty 判断集合是否为空
size 返回集合中元素的个数

Collection 代码示例

遍历

对于 Collection 集合，有三种遍历方式：分别是：迭代器、增强 for 和 Lambda。

迭代器

迭代器是集合的一种遍历方式，它提供了一种顺序访问集合元素的方式。迭代器提供了hasNext()和next()方法，用于判断集合中是否有下一个元素，并返回下一个元素。迭代器提供了remove()方法，用于删除当前元素。在迭代过程中，不能直接修改集合中的元素，只能通过迭代器进行修改。因为迭代器在初始化的时候会记录集合的当前的修改次数，每次调用 next 方法时，都会检查集合的修改次数是否与迭代器记录的的修改次数相同，当集合被修改时，集合中记录的修改次数就会发生改变，与迭代器中记录的次数不同，则抛出 ConcurrentModificationException 异常。

方法名 描述
hasNext() 判断是否有下一个元素
next() 获取下一个元素
remove() 删除当前元素
  1. 迭代器只能使用一次，每次使用都需要重新获取迭代器对象
  2. 迭代器在遍历过程中不能通过集合的 remove 方法删除元素，必须通过迭代器的 remove 方法删除元素。
  3. 迭代器是单向遍历的，只能向前遍历，不能向后遍历。

增强 for

增强 for 其实是语法糖，底层也是使用迭代器实现的。不仅可以遍历 Collection 集合，还可以遍历数组。

  1. 使用增强 for 遍历集合时，如果修改了 item 的值，集合中元素不会改变，因为 item 是集合中元素的一个副本。

Lambda

基于 Collection 接口的 forEach 方法，可以使用 Lambda 表达式遍历集合。底层使用的是迭代器实现的。ArrayList 集合除外，ArrayList 集合的 forEach 方法底层使用的是索引 for 实现的。

数据结构

数据结构是计算机科学中重要的概念，它描述了数据如何存储和操作。Java 中常用的数据结构有数组、链表、栈、队列、树、图等。

栈

栈：栈是一种 Last In First Out（LIFO）的数据结构，它遵循后进先出原则，即先入栈的元素，先出栈。

队列

队列：队列是一种 First In First Out（FIFO）的数据结构，它遵循先进先出原则，即先入队列的元素，先出队列。

数组

数组：数组是一种有序的集合，它按照索引顺序存储元素。数组的元素在内存中是连续的，因此可以快速访问任意位置的元素。但是对于插入和删除元素，数组效率较低。（查询快，增删慢）

链表

链表：链表是一种数据结构，它通过指针链接在一起的元素。链表在内存中不连续，因此不能通过索引访问元素。链表可以快速插入和删除元素，但是对于查询效率较低。（查询慢，增删快）

树

树：树是一种 hierarchical（层次）的数据结构，它通过父子节点的关系来将节点进行连接，组成一个树形结构。

  1. 一个节点只有一个父节点，而一个父节点可以有多个子节点
  2. 子节点的个数称为度
  3. 最顶层的节点称为根节点，根节点没有父节点。
  4. 度为 0 节点称为叶子节点。

二叉树

二叉树是一种树形结构，它每个节点最多有两个子节点。

二叉搜索树

二叉搜索树是一种二叉树，它满足以下条件：若每个节点的左子节点小于本身，且右子节点大于本身，则该树为二叉搜索树。

二叉树的遍历方式

  1. 前序遍历：根节点 -> 左子树 -> 右子树
  2. 后续遍历：左子树 -> 根节点 -> 右子树
  3. 中序遍历：左子树 -> 根节点 -> 右子树
  4. 层序遍历：从根节点开始，依次访问每一层的节点。

二叉搜索树的弊端

因为二叉搜索树要求节点的左子节点小于根节点，右子节点大于根节点，那么在添加元素时，若根节点太小，而后续添加的元素都比根节点大，就会导致后续的元素都添加到右子树，这样就会导致树的不平衡，导致查询效率低下。如下：

平衡二叉树

平衡二叉树是一种二叉树，它的每个节点的左右子树的高度差不超过 1。平衡二叉树是二叉树通过旋转机制来达到平衡的。旋转又分为左旋和右旋两种。当添加元素时，若左右子树的高度差超过 1，则需要进行旋转。通过旋转机制，是二叉树始终处于平衡状态。

旋转的应用场景：

  1. LL：当添加的元素添加到左子树的左子节点时，则需要将整体进行一次右旋
  2. LR：当添加的元素添加到左子树的右子节点时，则需要先将左子树进行一次左旋，变成LL，再将整体进行一次右旋
  3. RR：当添加的元素添加到右子树的右子节点时，则需要将整体进行一次左旋
  4. RL：当添加的元素添加到右子树的左子节点时，则需要先将右子树进行一次右旋，变成RR，再将整体进行一次左旋

平衡二叉树的弊端

平衡二叉树的查询效率比二叉搜索树高，但是每次添加元素时，都需要进行旋转，旋转的效率较低。由此诞生了红黑树。

红黑树

红黑树是一种自平衡的二叉搜索树，它的每个节点都有颜色，颜色分为红色和黑色。红黑树并不是高度平衡的，但是红黑树是自平衡的，通过红黑规则实现。

红黑规则

  1. 每个节点都是黑色或者红色
  2. 根节点是黑色
  3. 如果一个节点是红色，则它的父节点和两个子节点都是黑色
  4. 不存在连续两个连续的节点都是红色
  5. 所有叶子节点都是黑色
  6. 任意节点到叶子节点的所有路径上，都包含相同数量的黑色节点

红黑树插入节点的颜色处理规则

注意：以下规则仅为插入操作中部分情形的处理方式，完整实现还需考虑更多细节及连锁反应。

节点状态 颜色处理说明
默认 新插入的节点默认为红色，目的是尽可能不破坏原有的黑高路径
根节点 若新节点成为根节点，则必须将其颜色设置为黑色
黑色父节点 插入红色子节点不会违反任何红黑树性质，因此不需要进行任何调整
红色父节点 + 红色叔节点 - 父节点 → 黑色
- 叔节点 → 黑色
- 祖父节点 → 红色（如果祖父是根节点，则再变回黑色）
- 然后递归向上检查祖父节点
红色父节点 + 黑色叔节点 + 当前节点在右子树 此时需要进行左旋操作，使结构转换为 LL 型，然后再进一步处理
红色父节点 + 黑色叔节点 + 当前节点在左子树 - 父节点 → 黑色
- 祖父节点 → 红色
- 对祖父节点进行右旋操作完成结构调整

ArrayList

ArrayList 是基于数组实现的，默认容量为 0，当首次添加元素时，则会将数组的容量扩大为 10，后续再次添加元素时，若数组容量不够，会自动扩容。扩容后的大小为原容量的 1.5 倍。但如果扩容后仍不足以容纳所有元素，则会扩容到实际需要的大小。

LinkedList

LinkedList 是基于链表实现的，它不支持索引访问，但提供了双向链表和循环链表的实现。双向链表即每个节点都有前驱和后继，循环链表即最后一个节点的后继是第一个节点，第一个节点的前驱是最后一个节点。在添加元素时，会创建新的节点，并将新节点链接到链表的末尾。在删除元素时，会删除指定节点，并将指定节点的前驱和后继节点链接起来。

Vector

Vector 与 ArrayList 类似，也是基于数组实现的，但它是线程安全的。Vector 的扩容策略是每次扩容为原容量的 2 倍，而 ArrayList 是 1.5 倍。由于 Vector 的同步特性，在单线程环境下性能不如 ArrayList。

泛型

泛型（Generics）是一种类型参数化的抽象数据类型，它可以用来创建类型安全的、可复用的代码。泛型使得代码更加灵活，可以在运行时检查类型错误，从而避免了运行时错误。

Java 中的泛型其实是伪泛型，因为泛型是编译时特性，而 Java 是运行时特性，所以泛型在编译时会被擦除。可以通过反射机制在运行时向集合中添加泛型以外的元素。

Set

Set 集合的特点是：无序、不可重复、无索引。常用的实现类有 HashSet、TreeSet 和 LinkedHashSet。Set 集合继承自 Collection 接口，所以也继承了 Collection 的所有方法。

HashSet

HashSet 是基于哈希表实现的，查询速度快，适合存储大量数据。特点是无序、不重复、，无索引。

哈希表：JDK 8 以前，是数组 + 链表的实现方式，JDK 8 之后，是数组 + 链表 + 红黑树的实现方式。在添加数据时，通过计算该数据的哈希值，将数据存储在数组中，如果数组位置已经存在数据，则将数据存储在链表中。

哈希值：哈希值是通过hashCode()方法计算得到的，该方法在 Object 类中定义，所以所有类都继承了该方法。该方法默认使用对象地址计算哈希值，所以对象地址相同，哈希值也相同。但是一般情况下，都会重写 hashCode()方法，利用对象属性计算哈希值，对象属性相同，哈希值也相同。尽管如此，仍有极小概率，在地址不同，属性值不同的情况下，哈希值相同，我们称之为哈希碰撞。

哈希值代码示例

存储原理

HashSet 存储原理：在对象被创建时，会创建一个默认长度为 16 的数组，默认负载因子为 0.75，当数组的元素个数达到数组长度的 0.75 时，数组扩容为原来的2倍。每个数组元素都是一个链表，哈希值重复的元素会存储在链表中，当链表长度超过8，且数组长度超过 64 时，链表会变成红黑树。

添加元素：在添加元素时，会先通过 hashCode 方法计算出哈希值，然后再通过 (数组长度 - 1) & 哈希值 获取数组索引，将元素存储在数组中。当数组索引位置已存在元素，会通过 equals 方法判断元素是否相同，如果相同，则不添加，不同则添加。但是对于不同版本的 JDK，添加方式不一样，JDK 8 以前，使用的是头插法，JDK 8 之后，使用的是尾插法。

LinkedHashSet

LinkedHashSet 继承自 HashSet，在其基础上新增了一条链表，用于记录元素的插入顺序。因此具备了有序的特点。

存储原理

存储原理同 HashSet 无异，只不过添加元素时，会通过双向链表记录元素的插入顺序。每个元素都会记录前一个元素和下一个元素，这样通过链表就可以实现元素的插入顺序。

TreeSet

TreeSet 是基于红黑树实现的，因此其具备有序的特点，但是这个顺序并不是插入顺序，而是默认为升序排列。

添加元素：因为 TreeSet 是有序的，所以在添加元素时，会将该元素与其他元素比较，来确认该元素的顺序，所以在添加自定义类型的元素时，必须声明比较规则，有两种实现方式：1. 自定义类实现 Comparable 接口，并重写 compareTo 方法。2，在构造 TreeSet 对象时，通过 Comparator 参数指定比较规则。

Map

Map 接口是所有双列集合的根接口，常用的实现类有 HashMap、TreeMap 和 LinkedHashMap。

  • HashMap：基于哈希表实现，查询速度快，适合存储大量数据。
  • TreeMap：基于红黑树实现，支持按键排序，适合需要有序存储的场景。
  • LinkedHashMap：基于哈希表和链表实现，保持插入顺序，适合需要有序遍历的场景。
