Java 集合进阶

集合的体系结构

集合分为两大类：Collection 和 Map，Collection 为单例集合，其中只能保存值，而 Map 为双列集合，保存键值对。

Collection

Collection 接口是所有单例集合的根接口，常用的实现类有 List、Set 和 Queue。

  • List：有序可重复的集合，常用实现类有 ArrayList 和 LinkedList。
  • Set：无序不可重复的集合，常用实现类有 HashSet 和 TreeSet。
  • Queue：用于存储待处理元素的集合，常用实现类有 LinkedList 和 PriorityQueue。

常用方法

方法名 描述
add 添加元素
clear 清空集合
remove 删除元素
contains 判断是否包含某个元素
isEmpty 判断集合是否为空
size 返回集合中元素的个数

遍历

对于 Collection 集合，有三种遍历方式：分别是：迭代器、增强 for 和 Lambda。

迭代器

方法名 描述
hasNext() 判断是否有下一个元素
next() 获取下一个元素
remove() 删除当前元素
  1. 迭代器只能使用一次，每次使用都需要重新获取迭代器对象
  2. 迭代器在遍历过程中不能通过集合的 remove 方法删除元素，必须通过迭代器的 remove 方法删除元素。
  3. 迭代器是单向遍历的，只能向前遍历，不能向后遍历。

增强 for

增强 for 其实是语法糖，底层也是使用迭代器实现的。不仅可以遍历 Collection 集合，还可以遍历数组。

  1. 使用增强 for 遍历集合时，如果修改了 item 的值，集合中元素不会改变，因为 item 是集合中元素的一个副本。

Lambda

基于 Collection 接口的 forEach 方法，可以使用 Lambda 表达式遍历集合。底层使用的是迭代器实现的。ArrayList 集合除外，ArrayList 集合的 forEach 方法底层使用的是索引 for 实现的。

List

List 集合的特点是：有序、可重复、有索引。常用的实现类有：ArrayList、LinkedList 和 Vector。

常用方法

因为 List 继承自 Collection，所以 List 也拥有 Collection 的所有方法，除此之外，List 还新增了一些方法。

方法 描述
add(int index, E element) 在指定位置插入元素
remove(int index) 删除指定位置的元素
set(int index, E element) 修改指定位置的元素
get(int index) 获取指定位置的元素

数据结构

数据结构是计算机科学中重要的概念，它描述了数据如何存储和操作。Java 中常用的数据结构有数组、链表、栈、队列、树、图等。

  1. 栈：栈是一种 Last In First Out（LIFO）的数据结构，它遵循后进先出原则，即先入栈的元素，先出栈。
  2. 队列：队列是一种 First In First Out（FIFO）的数据结构，它遵循先进先出原则，即先入队列的元素，先出队列。
  3. 数组：数组是一种有序的集合，它按照索引顺序存储元素。数组的元素在内存中是连续的，因此可以快速访问任意位置的元素。但是对于插入和删除元素，数组效率较低。（查询快，增删慢）
  4. 链表：链表是一种数据结构，它通过指针链接在一起的元素。链表在内存中不连续，因此不能通过索引访问元素。链表可以快速插入和删除元素，但是对于查询效率较低。（查询慢，增删快）

ArrayList

ArrayList 是基于数组实现的，默认容量为 0，当首次添加元素时，则会将数组的容量扩大为 10，后续再次添加元素时，若数组容量不够，会自动扩容。扩容后的大小为原容量的 1.5 倍。但如果扩容后仍不足以容纳所有元素，则会扩容到实际需要的大小。

LinkedList

Vector

Vector 与 ArrayList 类似，也是基于数组实现的，但它是线程安全的。Vector 的扩容策略是每次扩容为原容量的 2 倍，而 ArrayList 是 1.5 倍。由于 Vector 的同步特性，在单线程环境下性能不如 ArrayList。

Set

Set 集合的特点是：无序、不可重复、无索引。常用的实现类有 HashSet、TreeSet 和 LinkedHashSet。

Map

Map 接口是所有双列集合的根接口，常用的实现类有 HashMap、TreeMap 和 LinkedHashMap。

  • HashMap：基于哈希表实现，查询速度快，适合存储大量数据。
  • TreeMap：基于红黑树实现，支持按键排序，适合需要有序存储的场景。
  • LinkedHashMap：基于哈希表和链表实现，保持插入顺序，适合需要有序遍历的场景。
