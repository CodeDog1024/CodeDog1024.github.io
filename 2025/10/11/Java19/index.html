双列集合的特点

双列集合是一个将键映射到值的集合。其中保存两个值：键（key） 和 值（value），称之为键值对。其中键不能重复，值可以重复。

Map

常用方法

方法 描述
put(K key, V value) 将指定的键值对添加到 Map 集合中。如果添加的键已存在，则更新其对应的值。
remove(Object key) 根据指定的键，删除对应的键值对。
clear() 删除所有键值对。
containsKey(Object key) 判断 Map 集合中是否存在指定的键。
containsValue(Object value) 判断 Map 集合中是否存在指定的值。
isEmpty() 判断 Map 集合是否为空。
size() 返回 Map 集合中键值对的数量。

HashMap

HashMap 继承自 Map 接口，拥有 Map 接口的所有方法，底层通过哈希表（数组+链表+红黑树）实现，具有无序、键不重复、值可重复和无索引的特性。

再存入哈希表计算哈希值时，只是用键的 hashCode 方法计算哈希值，然后通过哈希值对数组长度取模，得到数组索引。所以如果键是自定义类则必须重写 hashCode 和 equals 方法，而值则不需要。

练习

创建一个 HashMap，key 为学生对象，value 为学生籍贯，同名同年龄的学生认为是同一个人。

某班级共 80 名学生，现有四个景点 A、B、C、D，使用 HashMap 统计投票数。

LinkedHashMap

LinkedHashMap 继承自 HashMap，它内部维护了一个双向链表，用来维护插入顺序。 所以其具备有序、不重复、无索引的特性，元素顺序与插入顺序一致。

LinkedHashMap 代码示例

TreeMap

TreeMap 实现了 Map 接口，同 TreeSet 底层实现原理一致，都是红黑树。按照 Key 的大小进行排序，默认是升序排序，也可以指定排序规则。所以其具备有序（从小到大）、不重复（key）、无索引的特性。

练习

key：商品 ID，value：商品名称，实现升序和降序排序。

key：学生对象，value：学生籍贯，按照年龄 > 姓名 排序。

有一个字符串，统一每个字符的个数

HashMap 源码解析

HashMap 底层实现原理：数组 + 链表 + 红黑树。

在 HashMap 中，数组 table 用于存放元素，默认长度是 16，扩容阈值为 0.75，即数组长度乘以 0.75 = 12，当数组中元素个数达到 12 时，会扩容。

在 HashMap 中，每一个键值对都是一个 Node 对象，其中包含了 hash、key、value、next 等属性。而当链表转为红黑树时，则会使用 TreeNode 节点对象，其中又包含了 parent、left、right、prev、red 等属性。

空参构造函数：什么都不做，甚至不创建数组，只是将扩容阈值设为 0.75。

添加键值对：当首次添加键值对时，才会创建数组

TreeMap 原码解析

TreeMap 是一个红黑树结构，用于存储键值对。其中每一个键值对都是一个 TreeNode 节点对象，该节点对象包含键、值、颜色、左子节点、右子节点、父节点等信息。

构造函数：TreeMap 默认构造函数，创建一个空的 TreeMap 对象，但不会指定比较器。此时就要求键对象必须实现 Comparable 接口，否则会抛出 ClassCastException 异常。或者使用带参构造函数，指定比较器。

可变参数

可变参数（Variable Arguments）是一种语法糖，底层其实就是一个数组，允许在方法调用时传递任意数量的参数。格式为：方法名(参数类型... 参数名)

注意事项：可变参数有且只能有一个，且必须放在参数列表的最后。

集合工具类 Collections

Collections 是 Java 集合框架中的一个工具类，提供了一系列静态方法，用于操作集合对象。

常用方法

方法名 描述
addAll(Collection<? super T> c, T… elements) 将多个元素添加到集合中
shuffle(List<?> list) 随机打乱列表的顺序
sort(List<? extends T> list) 对列表进行排序
reverse(List<?> list) 反转列表的顺序
max(Collection<? extends T> coll) 返回集合中的最大元素
min(Collection<? extends T> coll) 返回集合中的最小元素
binarySearch(List<? extends Comparable<? super T>> list, T key) 二分查找
fill(List<? super T> list, T obj) 用指定对象填充列表
copy(List<? super T> dest, List<? extends T> src) 将源列表的元素复制到目标列表中
replaceAll(List<? super T> list, T oldVal, T newVal) 替换列表中的所有元素
swap(List<?> list, int i, int j) 交换列表中的两个元素
rotate(List<?> list, int distance) 旋转列表中的元素
disjoint(Collection c1, Collection c2) 判断两个集合是否没有元素相同
frequency(Collection<?> c, Object o) 返回集合中指定元素的出现次数
min(Comparator<? super T> comp, Collection<? extends T> coll) 返回集合中的最小元素，使用指定的比较器
