异常

异常是程序运行时出现的错误，当异常发生时，会终止程序的运行，并抛出一个异常对象。异常是不可避免的，我们无法消除异常，我能只能正确的处理异常。

异常体系

Throwable 是 Java 语言中所有异常的父类，子类有：Error、Exception。

Error 是错误，是系统层面的错误，这不是我们程序员需要处理的。

Exception 才是 异常，是程序员需要处理的。异常共分为两大类，分别是：运行时异常和编译时异常。

运行时异常

运行时异常：RuntimeException 及其子类， 是在运行时出现的异常，比如：空指针异常、数组越界异常、类型转换异常等等。这些异常在开发中是无法预见的。编译器无法检测到这些异常，所以不会提醒。

编译时异常

运行时异常：Exception，除 RuntimeException 以外的所有异常类， 是在编译时出现的异常，比如：文件不存在异常、类不存在异常、方法参数不匹配异常等等。这些异常在开发中是可以预见的。编译器会检查这些异常，如果这些异常可能会出现，编译器会报错，提醒我们处理这些异常。

异常代码示例

作用

  1. 提醒开发人员代码中可能会出现的 BUG，提前预防并处理，避免运行时程序崩溃。
  2. 作为错误信息，开发员人可以根据异常信息，快速定位问题所在并解决。

异常的处理

JVM 默认处理

将异常信息打印在控制台，并终止程序运行。

try-catch

我们可以使用 try-catch 来自己处理异常，当 try 中的代码发生异常时，会自动将异常对象传递给 catch，此时 catch 中的代码就会执行，且不会终止程序运行，后续代码会继续执行。

注意事项

  1. catch 方法的参数是一个异常对象，这个异常对象的类型必须和 try 中发生的异常类型一致，或者它的父类。否则当异常发生时，不会被捕获。
  2. 若 try 中没有异常发生，则 catch 中的代码不会执行。
  3. 若 try 中可能发生多种异常，那么可以写多个 catch 块来处理不同的异常，或者在一个 catch 块中处理多个异常，多个异常用 | 分隔开。

异常常用方法

方法名 描述
getMessage() 获取异常信息
printStackTrace() 打印异常信息
toString() 获取异常信息

抛出异常

throws 关键字，写在方法声明中，表示当前方法可能抛出指定的异常。当调用者调用该方法时，必须对抛出的异常进行处理。

thorw 关键字，写在方法中，表示当前方法主动抛出异常。抛出异常后，程序会终止。
