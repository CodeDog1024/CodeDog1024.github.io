网络编程

网络编程是指通过因特网（Internet）实现计算机之间的数据传输。

常见软件架构：

  1. C/S: 客户端/服务端
  2. B/S: 浏览器/服务器

网络编程三要素

  1. IP 地址：唯一标识计算机的数字地址。
  2. 端口号：端口号，标识计算机上的进程。
  3. 通信协议：描述计算机之间如何通信，如：TCP、UDP、http、ftp、smtp、pop3、imap 等。

IP

IP 地址（Internet Protocol Address）是 Internet 协议栈中的重要组件，用于唯一标识 Internet 中的计算机。IP 分为 IPV4 和 IPV6 两种。

  1. IPV4 地址格式为：A.B.C.D，其中 A、B、C、D 为 0-255 的数字。
  2. IPV6 地址格式为：ABCD:ABCD:ABCD:ABCD:ABCD:ABCD:ABCD:ABCD，其中 A、B、C、D 为 0-65535 的数字。

本机 IP 地址可以通过 ipconfig 命令查看，但是这个 IP 地址是本机的局域网 IP 地址，不是外网 IP 地址。当我们切换局域网时，会被重新分配新的 IP 地址。我们也可以通过 127.0.0.1 或者 localhost 来访问本机，这两个 IP 地址称为本地回环地址，永远不会被改变。

InetAddress 类

端口

端口（Port）是计算机网络中的一个数字，用于标识计算机上的进程。端口号范围：0-65535。

协议

协议（Protocol）是计算机网络中的重要概念，用于描述计算机之间如何通信。协议定义了计算机如何进行数据传输，以及如何处理数据。

模型

  1. OSI 模型：应用层 –> 表示层 –> 会话层 –> 传输层 –> 网络层 –> 数据链路层 –> 物理层
  2. TCP/IP 模型：应用层 –> 传输层 –> 网络层 –> 物理链路层

通信协议

  1. UDP: 无连接协议，速度快，单次最大传输 64KB，数据不安全，易丢失。
  2. TCP: 有连接协议，传输速度慢，无大小限制，数据安全。

UDP

UDP 是一种无连接的协议，它不建立连接，只发送数据，不保证数据安全。

UDP 有三种通信方法：单播、组播、广播。

  1. 单播：发送方发送数据给指定的接收方。
  2. 组播：发送方发送数据给多个接收方。
  3. 广播：发送方发送数据给所有接收方。

单播

单播需要指定具体的 IP 地址和端口号。

TCP

TCP 是一种有连接的协议，它建立连接，发送数据，并确保数据安全。

三次握手、四次挥手

TCP 三次握手是 TCP 连接的初始阶段，它用于建立连接。

  1. 客户端发送一个 SYN 包给服务端，表示要建立连接。
  2. 服务端收到 SYN 包后，发送一个 SYN+ACK 包给客户端，表示服务端已收到连接请求，并准备建立连接。
  3. 客户端收到 SYN+ACK 包后，发送一个 ACK 包给服务端，表示客户端已收到连接请求，并准备建立连接。

TCP 四次挥手是 TCP 断开的最后阶段，它用于断开连接。

  1. 客户端发送一个 FIN 包给服务端，表示要断开连接。
  2. 服务端收到 FIN 包后，发送一个 ACK 包给客户端，表示服务端已收到断开连接请求。
  3. 客户端收到 ACK 包后，发送一个 ACK 包给服务端，表示客户端已收到断开连接请求。
  4. 服务端收到 ACK 包后，发送一个 FIN 包给客户端，表示服务端已收到断开连接请求。
