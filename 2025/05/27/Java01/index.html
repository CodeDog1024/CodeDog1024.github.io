人机交互-图形化界面的小故事

  • 计算机刚刚出现的时候，并没有现在的图形化操作界面，而是在命令行下进行操作，通过在命令行中输入命令来操作计算机完成某些任务。
  • 随后，Xerox 公司发布了第一款具有图形化操作界面的计算机，名叫 Xerox Star，之后苹果、微软等公司也相继发布了具有图形化操作界面的计算机。
  • 图形化操作界面虽然让我们觉得操作电脑更加方便，但是也使得计算机的效率下降，因为需要额外加载一些资源。
  • 虽然目前的电脑都支持图形化操作界面，但是也保留了命令行，比如 Windows 的 CMD（命令提示符） 或者 PowerShell

打开 CMD

  • Windows + R -> 输入 cmd -> Enter

常见的 CMD 命令

命令 说明 举例
盘符名 + : 切换盘符 D:
dir 查看当前路径下的内容 dir
cd 路径 切换路径 cd D:\test\demo
cd .. 回退到上一级目录 cd ..
cd \ 切换到根目录 cd \
cls 清屏 cls
exit 退出 CMD exit

练习-利用 CMD 打开 QQ 并配置环境变量

利用 CMD 打开 QQ

  • 找到 QQ.exe 文件的所在文件夹
  • 打开 CMD ，使用 cd 路径进入 QQ.exe 文件的所在文件夹
  • 输入 QQ.exe 启动 QQ

环境变量

  • 上述操作中，要求必须在 QQ.exe 文件所在文件夹中执行，在其他路径下执行会提示错误，如果每次都要进入到这个文件夹中，那么就会很麻烦，所以可以配置环境变量，将 QQ.exe 文件所在文件夹添加到环境变量中，在任意路径下都可以执行 QQ.exe 启动 QQ

配置环境变量

  • 右键此电脑 -> 属性 -> 高级系统设置 -> 环境变量 -> 系统变量 -> Path -> 编辑 -> 新建 -> 输入 QQ.exe 文件所在文件夹的路径 -> 确定
  • 再次打开 CMD，输入 QQ.exe，就可以启动 QQ

为什么需要配置环境变量

  • 通过配置环境变量，就可以在任何路径下执行对应的命令打开指定程序或者资源

Java 学习-JDK 下载和安装

JDK 下载和安装

  • JDK 下载
  • 根据自己电脑的版本选择适合的版本进行下载安装
  • 安装路径不要包含中文、空格或者特殊符号

目录

  1. bin：Java 的执行文件
  2. conf： Java 的配置文件
  3. include：Java 的头文件
  4. jmods：Java 的模块文件
  5. legal：Java 的法律文件
  6. lib：Java 的库文件
