人机交互-图形化界面的小故事

  • 计算机刚刚出现的时候，并没有现在的图形化操作界面，而是在命令行下进行操作，通过在命令行中输入命令来操作计算机完成某些任务。
  • 随后，Xerox 公司发布了第一款具有图形化操作界面的计算机，名叫 Xerox Star，之后苹果、微软等公司也相继发布了具有图形化操作界面的计算机。
  • 图形化操作界面虽然让我们觉得操作电脑更加方便，但是也使得计算机的效率下降，因为需要额外加载一些资源。
  • 虽然目前的电脑都支持图形化操作界面，但是也保留了命令行，比如 Windows 的 CMD（命令提示符） 或者 PowerShell。

打开 CMD

  • Windows + R -> 输入 cmd -> Enter

命令 说明 举例

cd 路径 切换路径 cd D:\test\demo
cd .. 回退到上一级目录 cd ..
cd \ 切换到根目录 cd \
cls 清屏 cls
exit 退出 CMD exit

练习-利用 CMD 打开 QQ 并配置环境变量

利用 CMD 打开 QQ

  • 找到 QQ.exe 文件的所在文件夹
  • 打开 CMD ，使用 cd 路径进入 QQ.exe 文件的所在文件夹
  • 输入 QQ.exe 启动 QQ

环境变量

  • 上述操作中，要求必须在 QQ.exe 文件所在文件夹中执行，在其他路径下执行会提示错误，如果每次都要进入到这个文件夹中，那么就会很麻烦，所以可以配置环境变量，将 QQ.exe 文件所在文件夹添加到环境变量中，在任意路径下都可以执行 QQ.exe 启动 QQ。

配置环境变量

  • 右键此电脑 -> 属性 -> 高级系统设置 -> 环境变量 -> 系统变量 -> Path -> 编辑 -> 新建 -> 输入 QQ.exe 文件所在文件夹的路径 -> 确定
  • 再次打开 CMD，输入 QQ.exe，就可以启动 QQ。

为什么需要配置环境变量

  • 通过配置环境变量，就可以在任何路径下执行对应的命令打开指定程序或者资源。

  • Java 程序是无法直接运行的，需要先将 Java 程序编译成字节码文件，然后再通过 Java 虚拟机（JVM）来执行字节码文件。

Java 学习-常见小问题

BUG

  • BUG 是指程序中的错误或者缺陷，可能导致程序无法正常运行或者输出错误的结果。

常见问题

  1. 中英文符号问题：在编写程序时，必须使用英文符号。
  2. 单词拼写错误：Java 是区分大小写的，所以在编写程序时，必须注意单词的拼写和大小写。

Java 学习-环境变量

自动配置

  • JDK 在安装是会自动配置环境变量，所以我们不需要手动配置，但是只包含了四个工具命令：javac、java、javaw 和 jshell，如果需要使用其他工具命令，就需要额外手动配置环境变量。

手动配置

  • 新建一个系统变量，变量名为 JAVA_HOME，变量值为 JDK 的安装路径。。
  • 在 Path 系统变量中添加 %JAVA_HOME%\bin。

Java 学习-Java 能干什么

Java SE

Java 学习-Java 跨平台原理

高级语言的分类

  1. 编译型：如 C 语言，需要将源文件编译成可执行文件，然后直接运行。
  2. 脚本型：如 Python，无需编译，执行的时候逐行解释，逐行执行。
  3. 混合型：如 Java，需要先将源文件编译成字节码文件，然后再逐行解释运行。

跨平台原理

  • Java 程序并不是直接运行在计算机当中的，而是运行在一个名为 JVM 的虚拟机中，这样，只要要提供不同平台的 JVM，就可以在不同的平台上运行 Java 程序。

JDK 和 JRE

JDK（Java Development Kit）

  • JDK（Java Development Kit）是 Java 的开发工具包，包含了 JVM、核心类库、开发工具，用于开发 Java 程序。

JRE（Java Runtime Environment）

  • JRE（Java Runtime Environment）是 Java 的运行环境，包含了 JVM 和核心类库，用于运行 Java 程序。

三者关系

  1. JDK = JRE + 开发工具
  2. JRE = JVM + 核心类库
