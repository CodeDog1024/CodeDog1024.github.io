一、从技术选型开始

发表于2025-05-13|更新于2025-05-27
|浏览量:

项目概览

  • 基于 React + Spring Boot + Picocli + 对象存储的代码生成器共享平台
  • 分为三个阶段
    1. 第一阶段：制作本地代码生成器
    2. 第二阶段：开发代码生成器制作工具
    3. 第三阶段：开发在线代码生成器平台

项目背景

学习的意义

  1. 教程资料少
  2. 新颖且亮眼
  3. 技术亮点多

解决问题

  1. 自动生成常见的、重复性的代码片段
  2. 快速定制属于自己的代码生成器
  3. 一个平台维护多个不同的代码生成器

实际应用

  • 略

需求分析

调研

  • 前端：Ant Design Pro
  • 后端：Mybtais X IDEA

功能梳理

  1. 第一阶段：制作本地代码生成器
  • 能够通过执行程序生成代码
  • 能够通过命令行交互生成代码
  1. 第二阶段：开发代码生成器制作工具
  • 自动构建打包生成器
  • 支持通过配置制作生成器
  • 支持快速制作代码模板
  1. 第三阶段：开发在线代码生成器平台
  • 生成器检索查看
  • 在线发布生成器
  • 在线使用生成器
  • 在线制作生成器

技术选型

前端技术选型

  1. React
  2. Ant Design Pro
  3. Ant Design
  4. Ant Design Procomponents
  5. OpenAPI
  6. ESLint + Prettier + TypeScript

后端技术选型

  1. Spring Boot
  2. MySQL
  3. MyBatis-Plus
  4. Maven
  5. Picocli
  6. FreeMarker
  7. Redis
  8. XXL-JOB
  9. COS
  10. Hutool + Caffeine + Vert.x

部署技术选型

  1. Linux CentOS(这里我用的是 Ubuntu)
  2. 宝塔面板
  3. Nginx

其他开发工具

  1. IDE
  2. Git
  3. Apache JMeter

方案设计

代码生成器的核心原理

  1. 存在如下模板
  1. 传入参数
  1. 生成代码

第一阶段：制作本地代码生成器

阶段目标

  • 目标：做一个本地的代码生成器

业务流程

  1. 制作原始代码
  2. 基于原始代码，预设参数，编写动态模板
  3. 制作可交互的命令行工具，支持用户输入参数
  4. 得到代码生成器 jar 包，执行程序得到完整代码

实现思路

  1. 先事项静态代码的生成
  2. 根据参数制作模板
  3. 通过命令行交互工具，根据参数动态生成代码
  4. 封装 jar 包

关键问题

  • 略

第二阶段：开发代码生成器制作工具

阶段目标

  • 目标：做一个本地的代码生成器制作工具

业务流程

  1. 制作原始代码
  2. 基于原始代码，使用代码生成器制作工具，快速预设参数，编写动态模板
  3. 使用代码生成器制作工具，动态生成代码生成器 jar 包
  4. 得到代码生成器 jar 包，执行程序得到完整代码

实现思路

  1. 使用独立空间来存储和管理原始文件，模板文件
  2. 使用配置文件来记录参数和模板信息，自定义配置等
  3. 代码生成器制作工具需要具有多种可单独或组合使用的功能

关键问题

  • 略

第三阶段：开发在线代码生成器平台

阶段目标

  • 目标：打造一个在线代码生成器平台

业务流程

  1. 获取原始代码
  2. 基于原始代码，使用在线代码生成器制作工具，快速制作代码生成器
  3. 发布代码生成器至平台
  4. 在平台搜索代码生成器，下载使用

实现思路

  1. 使用 Web 开发框架实现代码生成器信息的增删改查
  2. 将本地配置和文件上云，存储到数据库，对象存储等云服务中
  3. 通过可视化界面来操作代码生成器制作工具
