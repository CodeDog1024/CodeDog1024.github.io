二、本地代码生成

发表于2025-05-13|更新于2025-05-16
|浏览量:

项目初始化

初始化根目录

  • 创建空项目 yuzi-generator

忽略无用提交

  • IDEA -> Settings -> Plugins
  • 搜索 .ignore 插件并安装，点击应用
  • 右键项目 -> New -> .ignore File -> .gitignore(Git)
  • 选中 Java 和 JetBrains
  • 右键 .gitignore 文件 -> Git -> Add
  • 建议新建 README.md

创建 Demo 示例代码工程

  • 新建 yuzi-generator-demo-projects 目录，将 ACM 模板项目赋值到该目录下
  • 如果是自己手动建目录的话记得在项目结构里把 src 文件夹设为源代码，不然无法识别
  • MainTemplate.java

创建本地代码生成器项目

  • 创建 yuzi-generator-basic 项目
    1. 存放在 yuzi-generator 目录下
    2. 取消勾选 Create Git repository
    3. 使用 Maven 管理项目
    4. JDK 选择 1.8(我这里用的 21)
    5. 在高级设置中，指定 GroupId 和 ArtifactId
  • 创建好后要在 新窗口 打开项目
  • 在 pom.xml 文件中引入依赖
    1. Hutool
    2. Apache Commons Collections4
    3. Lombok
    4. JUnit4
  • 导入依赖后刷新 Maven

实现流程

需求拆解

  • 把需求分为两部分
    1. 本地代码生成器
    2. 基于命令行的脚手架
  • 文件分类
    1. 静态文件
    2. 动态文件
  • 调用 main 方法时传入参数

实现步骤

  1. 生成静态文件
  2. 生成动态文件
  3. 同时生成静态文件和动态文件
  4. 开发命令行工具
  5. 封装 jar 包

静态文件生成

  • 本质就是文件的复制
    1. 使用现成的工具库
    2. 手动编写递归算法，复制所有目录和文件

使用现成的工具库复制目录

  • 在 com.yupi.generator 目录下新建 StaticGenerator 类
  • 编写静态方法 copyFilesByHutool
  • 使用 Hutool 中的 FileUtil.copy() 方法
  • 然后再编写一个 Main 方法来调用即可

递归遍历

  • 可参照 Hutool 的 FileUtil.copy() 源码

动态文件生成思路

明确动态生成需求

  1. 在代码开头添加作者 @Author 注释
  2. 修改代码中的提示内容
  3. 将循环输入改为单词输入

动态生成的核心原理

  1. 输入参数：author = yupi
  2. 模板如下：@author ${author}
  3. 将模板中的参数替换为输入的参数：@author yupi

FreeMarker 模板引擎入门及实战

什么是 FreeMarker

  • FreeMarker 是 Apache 的开源模板引擎

模板引擎的作用

  • 接收模板和 Java 对象，将模板中的参数替换为 Java 对象中的参数，输出完整内容

模板

  • FreeMarker 拥有自己的模板编写规则，使用 FTL 来表示 FreeMarker 语言，文件后缀为 ftl
    1. 文本：固定值
    2. 参数：${变量名}
    3. ftl 方法：<#xxx…>，比如 <#list elements as element> 实现循环
    4. 注释：<#–…–>

问题解决示例

  • 之前生成的文件中，传入的年份 2025 变成了 2,025，这是因为 FreeMarker 使用了 Java 平台的本地化敏感的数字格式信息，查阅官方文档，只需要在 configuration 中配置类的 number_format 格式即可

动态文件生成实现

  1. 定义数据模型
  2. 编写动态模板
  3. 组合生成
  4. 完善优化

定义数据模型

  • 在 com.yupi.model 下新家模板配置对象，用来接收要传递给模板的参数。

编写动态模板

  • 在 resources/templates 下新建 FTL 模板文件 MainTemplate.java.ftl，模板名要和数据模型名称一致
  • 在需要动态生成的地方使用 ftl 语法挖坑

组合生成

  • 同静态文件生成器一样，在 com.yupi.generator 目录下新建动态文件生成器类 DynamicGenerator，在按照之前的 FreeMarker Demo 实战一样编写代码

完善模板

  • 完善模板，给数据模型中的变量赋初值，可以在数据模型中设置，也可以在模板中使用 ftl 语法设置

抽取方法

  • 抽取方法，这样 Main 中的代码就可以大大简化了

异常处理

  • 异常处理，不要全部抛到外层处理

  • 组合调用 静态文件生成器 和 动态文件生成器，先生成静态文件，在生成动态文件覆盖
