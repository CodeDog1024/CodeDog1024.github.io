三、命令行开发

Java 命令行开发方案

  • 命令行程序简称CLI，是指通过命令行界面运行的应用程序，通过终端窗口接收用户输入的纯文本命令
  • 如：UNIX/Linux 的终端，Windows 的命令提示符和 Powershell 等

命令的结构

  1. command：命令
  2. option：选项
  3. parameter：参数

为什么要开发命令行

  1. 不依赖图形界面，轻量
  2. 可以在终端环境运行
  3. 可以和用户交互
  4. 操作快捷

命令行的作用

  • 使用命令行的方式与用户交互，通过用户输入的参数构建数据模型，根据用户要求生成对应的文件

实现方案

自主实现

  • 通过 Scanner 函数，配合 while、if...else... 等语句获取用户输入的文本，然后去解析其中的命令、选项、参数等信息

第三方库

  • Picocli、JLine、JCommander、Apache Commons CLI、Jopt Simple

Picocli 命令行框架入门

入门 Demo

  • 在 yuzi-generator-basic 中的 pom.xml 中引入依赖
  • 然后再 com.yupi 下新建 cli.example 包，用来存放示例代码，将官方快速入门的示例代码复制到 com.yupi.cli.example 下，并修改 run 方法中的代码
  • 修改主程序的执行参数 args 来测试程序 -s 12 hello world
  1. 创建一个实现 Runable 或 Callsble 接口的类，就是一个命令
  2. 使用 @Command 注解标记该类并为其命名，使用 mixinStandardHelpOptions 属性开启用户手册 --help 和 --version
  3. 通过 @Option 注解将字段设置为命令行选项，给选项设置名称和描述
  4. 通过 @Parameters 注解将字段设置为命令行参数，设置默认值和描述
  5. Picocli 会将命令行参数转为强类型值并自动注入到注解字段中
  6. 在类的 run 方法中定义业务逻辑
  7. 通过 CommandLine 对象的 execute 方法处理用户输入的命令
  8. CommandLine.execute 方法会返回一个结果，用于查看成功还是失败

帮助手册

  • 使用 @Command 注解，将 mixinStandardHelpOptions 属性设置为 true开启用户手册 --help 和 --version

命令解析

  • 两个核心注解
    1. @Option
    2. @Parameters
  • @Option
    1. names：指定选项的引文名称
    2. description：指定选项的描述信息
  • @Parameters
    1. paramLabel：参数标签，作用类似于描述信息
    2. defaultValue：默认值
    3. requird：必填
  • 此外，命令解析支持多值选项，只需要使用数组来接受参数即可

交互式输入

基本能力

  • Picocli 支持输入交互式输入，只需要将 @Option 注解的 interactive 设置为 true
  • 执行参数 -u root -p 后，会在控制台提示输入密码
  • 注意：在 Picocli 4.6 之后，当通过 jar 包的形式运行代码时，当用户在控制台输入密码时，为了输入的安全性，并不会显示出来，需要执行 echo 为 true 才可以，还可以通过 prompt 来设置提示语

多个选项交互式

  • Picocli 只是在一个命令中指定多个交互式输入，会按照顺序提示用户
  • 在上述基础上新增 checkPassword，同样开始交互式输入
  • 输入参数 -u root -p -cp 运行程序

可选交互式

  • 如果使用了 interactive 将选项指定为交互式输入，那么就无法在执行命令直接输入参数，如 -u root -p 123456 -cp，会出现参数不匹配的情况，这时可以通过可选交互式来选择是直接输入参数还是通过交互式输入参数
  • 通过 @Option 注解的 arity 属性来指定选项可接收的参数个数
  • 通过设置 arity = "0..1" 就可以让选项接收 0 ~ 1 个参数，此时输入 -u root -p 123456 -cp
    便不会报错，且不会提示用户输入密码，只需要出入 checkPassword 即可
  • 建议给所有选项都添加 arity = "0..1" ，这样用户可以选择直接输入参数还是交互式输入

强制交互式

  • 交互式的前提是用户必须在执行命令时输出对应的选项如 -p，程序才会提示用户输入密码，但是如果用户没有输入，则系统不会提示，且默认值为 null，此时就需要使用强制交互式
  1. 可以使用 @Option 注解的 required = true 属性，当用户没有输入选项时会提示报错
  1. 或者自定义业务逻辑，判断用户是否输入了选项，没有则提示用户输入
  1. 编写一段通用检测代码，用来检测用户输入的命令是否输入了必要的选项，若没有，则自动补充

子命令

  • 子命令是指命令中又包含一组命令，相当于命令的嵌套

声明式

  • 通过 @Command 注解的 subcommands 属性来给命令添加子命令

编程式

  • 在创建 CommandLine 对象时，调用 addSubcommand 方法来绑定子命令

实践

  • 编写一个示例程序，支持增、删、查三个子命令
  • 输入 --help 时会打印出所有的命令信息
  • 使用 Junit 测试
