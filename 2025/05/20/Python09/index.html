了解异常

什么是异常

  • 异常是程序运行过程中出现的错误，程序会因此停止运行，就是所谓的 BUG
  • 比如当我们想读取一个不存在的文件时，就会提示 FileNotFoundError 这就是异常

异常的捕获方法

为什么要捕获异常

  • 世界上没有完美的程序，为了保证程序在发生错误的同时依旧可以继续运行，所以我们需要尽可能的处理能够预知，且可以避免的错误，我们称之为捕获异常

捕获异常

  • 使用 try...except... 语句，将可能会发生异常的代码包含在 try 语句中，当运行时发生异常，except 语句会捕获这个异常，并执行 except 语句中的代码
  • 当我们能够预知异常的类型的时候，我们可以使用 except 异常类型 as e: 来捕获指定异常，比如读取不存在时，我们可以使用 except FileNotFoundError as e: 来捕获这个异常，再或者使用未定义的变量，使用 except NameError as e: 来捕获这个异常。
  • 但是这样子就不能捕获其他类型的异常了
  • 当需要捕获多个异常的时候，可以使用写在一个括号中，并使用逗号隔开
  • 捕获所有异常，使用Exception，其中Exception是所有异常的基类

else 语句

  • try 块没有异常发生时，会执行else块

finally 语句

  • try 块不管有没有异常发生，都会执行finally块

异常的传递性

  • 函数中如果发生异常，会向上传递，知道被捕获或者程序终止
  • 只需要在整个函数的调用过程当中，任意一个位置捕获异常，就可以解决

Python 模块

什么是模块

  • Python 中的模块，就是 Python 文件，其中包括编写好的变量和函数，其他文件可通过导入模块来使用该文件中的代码

模块的导入

  • Python 模块的导入方式：[from 模块名] import [模块 | 类 | 变量 | 函数 | *] [as 别名]
  • import 导入模块，需要使用 模块名 来调用模块中的函数、类、变量等。
  • 导入部分模块内的指定功能，或者使用 * 来导入所有功能。
  • 使用 as 语法为模块或者模块中的函数取别名

自定义模块

  • 自定义模块，就是创建一个 Python 文件，然后编写好代码，就可以在其他文件中导入使用
  • 新建文件 calc.py
  • 注意：当模块名或函数名等重名时，后导入的模块会覆盖先导入的模块

__name__ 变量

  • 在模块中主程序的代码在导入模块时会自动执行
  • 若不想在模块中执行代码，可以在模块中使用一个 __name__ 变量，当模块被导入时，便不会被执行

__all__ 变量

  • 在模块中定义一个 __all__ 变量，用来指定模块中哪些函数可以被导入使用，只针对 from xxx import * 起作用，对于其他导入语法不起作用

Python 包

  • 当 Python 模块过多时，为了方便管理，可以将其放在一个文件夹中，此时这个文件夹就叫做包，且该文件夹下需要包含一个名为 __init__.py 的文件

自定义包

  • 右键工程 -> 新建 -> Python 软件包
  • 会自动生成文件 __init__.py，然后手动新建模块 calc.py 和 io.py

导入包

  • 使用 import 包名.模块名 导入包，并且使用 包名.模块名.函数名 来调用模块中的函数
  • 使用 from 包名 import 模块名导入
  • __all__变量 同样也可以在包中使用，只不过需要写在 __init__.py 文件中
  • 简化导入，在导入包的时候，__init__.py 中的代码也会被执行，所以可以在 __init__.py 文件中导入需要的方法，这样子在导入包时就不需要再导入模块了

安装第三方 Python 包

  • 第三方包就是其他开发者创建的 Python 包，并且存放在 PyPI 上，我们可以通过 pip install 包名 来安装第三方包
